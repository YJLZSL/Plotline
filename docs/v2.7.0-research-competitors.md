# v2.7.0 竞品调研：Timeline 布局、AI 暴露与创作交互

> 调研日期：2026-06-27  
> 目标版本：Plotline v2.7+  
> 调研对象：Notion Timeline、Aeon Timeline、Claude、Kimi / 豆包 / 通义千问  
> 关注维度：动画/性能提示、AI 能力暴露方式、时间轴与叙事组织、游戏化/趣味性、可落地建议

---

## 一、快速对比

| 维度 | Notion Timeline | Aeon Timeline | Claude | Kimi / 豆包 / 通义千问 |
|---|---|---|---|---|
| **核心定位** | 数据库驱动的甘特图视图 | 专业叙事/项目时间轴 | 通用 AI 协作助手 | 中文写作/办公助手 |
| **时间轴模式** | 横向甘特条，按属性分组 | Timeline / 关系 / 地铁 / 叙述 / 大纲 / 思维导图 | 无原生时间轴 | 无原生时间轴 |
| **AI 入口** | 行内「Ask AI」+ `/` 命令 + Notion Agent | 无 AI | 输入框 + Projects + Memories + `/` 命令（Claude Code） | 输入框 `@` + 一键续写/润色/总结 chip + 浏览器插件 |
| **动画风格** | 极简、几乎无动效，重性能 | 桌面级原生拖拽，过渡克制 | 流式打字 + Artifact 展开 | 流式输出 + 复制/重试按钮淡入 |
| **上下文能力** | 全工作区 + 已连接应用搜索 | 项目内全实体关联 | Projects / Memories / 100万 token 上下文 | 超长上下文（Kimi 200万字 / 通义 1000万字文档） |
| **游戏化/趣味** | 无 | 无 | 无 | 豆包有 IP 化角色、Kimi 有「长文本」成就式口碑 |
| **对 Plotline 启示** | 把 AI 按钮放在上下文最近处；视图切换轻量 | 多视图切换是叙事刚需；自定义历法/相对时间 | 记忆与项目是长期创作的核心；Artifact 适合结构化产出 | 中文作者更爱「一键 XX」chip；超长上下文是差异化卖点 |

## 二、Notion Timeline

### 2.1 时间轴布局与交互
- **甘特式横向条**：每个数据库条目是一个可横向拖动的时间条，支持开始/结束时间两个独立属性，通过「Timeline By」切换依据属性。
- **分组与缩放**：按 Select / Person / Status 等属性垂直分组；顶部时间标尺支持日/周/月/季/年缩放。
- **拖拽调整**：直接拖动条的两端修改起止时间，拖动整段平移；无复杂依赖线，保持简洁。
- **与 Database 一体化**：Timeline 只是 Database 的一个 View，属性、筛选、排序、模板完全共享。

### 2.2 动画与性能
- 动画极为克制：条目移动以 150–200ms 的淡入/位移为主，无弹性效果。
- 长列表采用虚拟滚动 + 懒加载，大量条目时仍保持流畅。
- 视图切换（Table / Board / Timeline / Calendar）几乎是即时渲染，重性能而非过渡仪式感。

### 2.3 AI 能力暴露
- **行内 Ask AI**：在任意块末尾按空格或点击 sparkle 图标唤出「Ask AI」，可选「续写 / 总结 / 翻译 / 改语气 / 头脑风暴」。
- **斜杠命令**：输入 `/ai` 直接唤起 AI 菜单；支持自定义提示词保存为 slash command。
- **Notion Agent**：2026 年主推的「Custom Agents」可在后台自动执行任务（如整理会议纪要、归档任务），通过侧边栏或命令面板触发。
- **模型可选**：Business 版支持在设置中切换 GPT-4.1 / Claude 4 等模型。

### 2.4 对 Plotline 的启示
1. **AI 入口要贴近上下文**：不是只有在 AI 面板才能提问，而是在事件卡片、大纲节点、编辑器行内都能一键唤起。
2. **自定义 slash command**：把常用创作 SOP（如「生成角色小传」「检查时间线漏洞」）封装为 `/` 命令，减少重复输入。
3. **视图即筛选**：Timeline 是同一数据的不同透镜，Plotline 可强化「时间轴 ↔ 大纲 ↔ 角色」的双向同步。
4. **克制动画保证性能**：时间轴数据量大时，动画时长应控制在 200ms 以内，优先保证拖拽跟手。

## 三、Aeon Timeline

### 3.1 时间轴与叙事组织
- **多视图是核心卖点**：Timeline（时间线）、Relationship（关系）、Subway（地铁图）、Narrative（叙述）、Outline（大纲）、Mindmap（思维导图）、Spreadsheet（表格）。
- **自定义时间与历法**：支持绝对时间、相对时间，精度可到秒；可自定义一周天数、一天小时数，适合奇幻/科幻世界观。
- **角色/地点/事件关联**：每个事件可绑定角色、地点、故事单元，并在关系视图中直接创建和关联实体。
- **叙述视图（Narrative）**：按章节/幕手动拖入事件，决定「故事讲述顺序」，与按时间排序的 Timeline 解耦，方便处理闪回、倒叙、叙述性诡计。
- **外部写作工具同步**：可与 Scrivener、Ulysses 同步，事件变动双向提醒，避免「吃书」。

### 3.2 交互与视觉
- 桌面原生应用，拖拽调整时间节点和持续时间是核心交互。
- 地铁视图用类似地铁路线的方式展示多条故事线交叉，适合人物地点繁多的场景。
- 无显著动画过渡，依赖原生滚动和即时重绘，强调专业工具的稳定感。

### 3.3 AI / 写作辅助
- Aeon Timeline 本身**无 AI 功能**，但其与写作工具同步的 Workflow 提供了「结构化辅助」：通过时间轴约束创作，减少逻辑漏洞。

### 3.4 对 Plotline 的启示
1. **叙述视图与时间轴视图分离**：Plotline 已有时间轴，可考虑增加「叙事编排视图」，让作者按章节/幕组织事件，而不只是按时间。
2. **支持自定义历法/相对时间**：奇幻/科幻创作中，「故事内时间」与现实时间不同，自定义历法是高阶刚需。
3. **关系/地铁视图**：当角色与地点增多时，用「关系视图」和「地铁图」展示事件-角色-地点的交集，比纯时间轴更清晰。
4. **与外部编辑器同步**：未来可考虑与 Typora / Obsidian / Scrivener 同步大纲或事件，扩展生态。

## 四、Claude（Anthropic）

### 4.1 AI 能力暴露
- **输入框即入口**：主对话界面简洁，输入框下方提供快捷 prompt 胶囊（如 "Help me develop a unique voice" / "Improve my writing style"）。
- **Projects（项目）**：可创建工作区项目，上传文件、设定长期上下文，Claude 会跨对话记住项目记忆。
- **Memories**：Claude 自动记住用户偏好和重要事实，长期创作时可减少重复交代。
- **Artifacts**：在对话中生成可独立编辑的文档、代码、表格、 checklist 等结构化内容，支持版本对比与一键复制。
- **Claude Code / Cowork**：桌面端支持将任务委托给 Claude，Claude 可读取本地文件、调用工具并分步执行，用户审批后继续。
- **Slash Commands（Claude Code）**：支持 `/test`、`/pr-review`、`/explain` 等自定义命令，把 SOP 固化为一键触发。

### 4.2 上下文意识
- 100万 token 上下文窗口（2026 年 Claude Opus 4.7），可一次性吞下整本小说或代码库。
- 项目文件、历史对话、用户记忆共同构成上下文，Claude 会在需要时主动提问确认。
- 状态提示：模型选择器位于顶部，无持续状态灯，失败时以 toast 提示。

### 4.3 动画与性能
- 流式输出采用打字机效果 + 光标脉冲，底部自动平滑滚动。
- Artifact 展开有 200ms 左右的淡入/高度动画。
- 无游戏化或趣味性元素，整体风格冷静、专业。

### 4.4 对 Plotline 的启示
1. **记忆是长期创作的核心**：Plotline 的 AI 应记住作者偏好（如「常用语气」「世界观设定」），减少每轮重复 prompt。
2. **Artifact 式结构化输出**：AI 生成的大纲、角色卡、时间线补丁应以可编辑卡片/面板形式呈现，而非纯文本。
3. **项目级上下文**：把工作区作为 Claude Project 的对应概念，AI 默认基于当前工作区的全部事件、角色、地点回答。
4. **Slash command 固化 SOP**：把「检查时间线逻辑」「生成下章大纲」「补全世界观细节」等高频操作做成 `/` 命令。

## 五、Kimi / 豆包 / 通义千问（中文写作助手）

### 5.1 共同特点
| 产品 | 代表能力 | 写作入口 |
|---|---|---|
| **Kimi** | 200万字超长无损上下文、联网搜索、长思考 | 网页/App 输入框 + `@Kimi` |
| **豆包** | 多场景文本生成、中文语义深度理解、IP 化角色 | 网页/App + 桌面端划词工具栏 |
| **通义千问** | 1000万字文档解析、公文/汇报写作、多文档总结 | 网页/App + 浏览器插件 |

### 5.2 AI 能力暴露方式
-  **@ 召唤**：在输入框输入 `@Kimi` 并空格，即可让 AI 基于当前上下文执行任务。
- **一键动作 chip**：输入框上方或右侧提供「续写 / 润色 / 总结 / 扩写 / 翻译」等一键按钮，点击即执行。
- **划词工具栏（豆包桌面端）**：选中文字后弹出悬浮条，提供翻译、润色、改写、生成大纲等操作。
- **文档解析**：直接拖拽上传多个文档，AI 自动总结、对比、提取关键信息。
- **提示词模板**：社区大量分享「润色指令」「公文写作模板」「小说大纲模板」，降低使用门槛。

### 5.3 游戏化/趣味性
- **豆包**：有虚拟角色聊天、语音对话等娱乐化入口，写作之外提供情感陪伴。
- **Kimi**：以「200万字长文本」形成口碑标签，用户有「能一次性读完一本书」的成就感。
- **通义千问**：强调「百份文档一键解析」的效率爽感。

### 5.4 动画与性能
- 流式输出是标配，配合复制/重试/重新生成按钮的淡入。
- 文档上传后显示解析进度条，长文本处理时显示「正在阅读第 N 页」。
- 动画简单直接，不强调过渡效果，重点突出「快」和「能处理长文本」。

### 5.5 对 Plotline 的启示
1. **中文作者更爱「一键完成」**：相比 slash command，中文用户更习惯点击「续写」「润色」「总结」chip。Plotline 的 AI 面板应提供这类显式按钮。
2. **超长上下文是杀手锏**：Plotline 作为本地工具，可将整本小说、全部角色/地点作为上下文提供给 AI，这是云端通用助手难以匹敌的优势。
3. **划词/选中即操作**：在编辑器中选中一段文字后，弹出 AI 工具条（润色、改语气、插入到时间轴），减少打断创作流。
4. **提示词模板市场**：内置「小说大纲生成」「角色小传」「时间线漏洞检查」等模板，并允许用户保存自定义模板。
5. **进度可视化**：长文本处理或批量分析时，显示明确进度条，避免用户以为应用卡死。

## 六、综合建议：Plotline v2.7+ 可落地的方向

### 6.1 AI 交互层
1. **三处 AI 入口**：
   - 全局 AI 面板（已有）
   - 编辑器行内/划词工具条
   - 时间轴事件卡片右键菜单 + `/` 命令
2. **高频动作 chip**：在 AI 输入框上方固定「续写 / 润色 / 总结 / 检查逻辑 / 生成角色」等一键按钮。
3. **项目级记忆**：让 AI 记住当前工作区的设定、风格、常用角色关系，跨对话保持上下文。
4. **Artifact 卡片**：AI 生成的内容以可编辑卡片呈现，支持「插入到时间轴 / 插入到大纲 / 复制 / 弃用」。

### 6.2 时间轴与叙事层
1. **叙述视图（Narrative View）**：按幕/章节组织事件，支持拖拽调整叙事顺序，与时间轴视图解耦。
2. **关系/地铁视图**：为角色、地点、事件建立可视化关系图，帮助理清多线叙事。
3. **自定义历法/相对时间**：支持奇幻/科幻创作中的非标准时间单位。
4. **事件卡片分层**：标题 + 摘要 + tag 行，hover 展开详情，减少视觉拥挤。

### 6.3 动画与性能
1. 保持 200ms 以内的 ease-out 动画，长列表使用虚拟滚动。
2. AI 流式输出时底部平滑滚动，显示光标脉冲。
3. 长文本/批量处理时显示进度条，避免无反馈等待。

### 6.4 趣味与差异化
1. **本地超长上下文**：把「整本小说 + 全设定作为 AI 上下文」作为核心卖点。
2. **成就/统计**：参考番茄钟已有统计，可扩展「本月创作字数」「完成事件数」「AI 辅助次数」等轻量成就。
3. **中文创作模板**：内置网文/剧本/轻小说常用模板，降低首次使用门槛。
