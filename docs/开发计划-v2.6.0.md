# Plotline v2.6.0 开发计划

> 制定日期：2026-06-26  
> 依据：`docs/审计报告-v2.5.4.md`、`docs/市场调研报告-v2.6.0.md`

---

## 一、目标

在 v2.5.4 基础上，落地 **4 项 P1 增强**，显著提升 AI 工作流效率、时间轴信息密度管理能力，以及新用户首次使用引导。

---

## 二、选定功能与优先级

| 优先级 | 功能 | 解决痛点 | Effort | UX 价值 | 技术风险 |
|---|---|---|---|---|---|
| P1 | AI 风格模板 chips | 用户每次输入长篇提示词，国产助手已教育市场一键切换风格 | 低 | 高 | 低 |
| P1 | AI apply-to-doc | AI 回复无法直接写入小说/笔记编辑器，需手动复制粘贴 | 中 | 高 | 中 |
| P1 | 时间轴筛选与折叠 | 事件密度高时轨道拥挤，缺少快速聚焦叙事线的能力 | 中 | 高 | 低 |
| P1 | 模板向导空状态 | 新用户创建项目后空画布无从下手 | 低 | 高 | 低 |

> 注：市场调研中原「AI 流式输出」在 v2.5.4 代码中已实际落地（`aiChatStream` + 消息区逐字渲染），故 v2.6.0 将 AI 重点调整为「风格模板」与「apply-to-doc」。

---

## 三、功能 1：AI 风格模板 chips

### 3.1 需求描述

在 AI 面板输入框上方增加横向滚动的风格/题材 chips，点击后自动填充输入框并附带对应 system prompt 风格。内置至少 6 种模板：

- 玄幻
- 都市
- 言情
- 悬疑
- 番茄风（爽文快节奏）
- 知乎风（第一人称/强钩子）

### 3.2 影响文件

- 新增：`src/features/ai/promptTemplates.ts`
- 修改：`src/components/layout/AiAssistantPanel.tsx`
- 修改：`src/i18n/locales/zh-CN.json`
- 修改：`src/i18n/locales/en.json`

### 3.3 实现要点

1. `promptTemplates.ts` 导出 `AI_STYLE_TEMPLATES` 数组，每项包含 `id`、`labelKey`、`icon`、`template`（用户可见提示词片段）、`systemPrompt`（追加到 AI 请求的系统提示词）。
2. 在 `AiAssistantPanel.tsx` 输入框上方新增 `<TemplateChips />` 组件。
3. 点击 chip 时：
   - 将 `template` 写入输入框；
   - 在发送时通过 `context` 的 `systemPromptOverride` 字段（如不存在则新增）传入 `systemPrompt`。
4. 当前处于流式输出或上下文收集中时禁用 chips。

### 3.4 验收标准

- [ ] 6 种以上模板在 AI 面板可见。
- [ ] 点击 chip 后输入框自动填充对应提示词片段。
- [ ] 发送后 AI 回复风格符合所选模板（可通过集成测试校验 system prompt 是否传入）。
- [ ] i18n 文案覆盖简体中文与英文。

## 四、功能 2：AI apply-to-doc

### 4.1 需求描述

在已有 apply-to-entity（创建新事件/角色/地点/大纲节点等）基础上，支持将 AI 生成的文本直接插入当前打开的编辑器：

- **Novel 视图**：插入当前章节光标处，或替换选中段落。
- **Notebook 视图**：插入当前笔记光标处，或替换选中段落。
- **Outline 视图**：追加/替换当前选中大纲节点的内容字段（如果大纲编辑器已支持富文本，则走编辑器插入）。

### 4.2 影响文件

- 修改：`src/types/ai.ts`（扩展 `AiInsertTarget`）
- 修改：`src/features/ai/hooks.ts`（`useApplyAiOutput`）
- 修改：`src/features/ai/api.ts`（如需要新增 apply 命令）
- 修改：`src/components/views/NovelView.tsx`
- 修改：`src/components/views/NotebookView.tsx`
- 修改：`src/components/views/OutlineView.tsx`
- 修改：`src/components/layout/AiAssistantPanel.tsx`（`APPLY_TARGETS` 增加 novel/notebook）
- 新增/修改：`src/stores/editorSelection.ts`（保存当前编辑器焦点与选区）

### 4.3 实现要点

1. 在 `types/ai.ts` 中新增 `AiInsertTarget`：
   - `'novel_chapter'`
   - `'notebook_content'`
   - `'outline_node_content'`
   - `'replace_selection'`（插入模式枚举）
2. 新增 `editorSelectionStore` 或扩展 `aiContextStore`，记录：
   - 当前激活的编辑器类型（novel/notebook/outline）
   - 当前编辑器实例引用或选区范围
3. `NovelView`/`NotebookView` 在编辑器获得焦点时向 store 注册自身 editor 实例；失焦时注销。
4. `useApplyAiOutput` 处理新 target：
   - 调用 editor 的 `insertContent` / `setContent` / `chain().focus().insertContentAt(...)` 等 TipTap API。
   - 若用户有选区，提供「替换选中」与「插入光标处」两个动作。
5. `AiAssistantPanel` 的 `APPLY_TARGETS` 增加 novel/notebook，并根据当前 view 推荐。

### 4.4 验收标准

- [ ] 在 Novel 视图中，AI 回复下方出现「插入当前章节」动作。
- [ ] 在 Notebook 视图中，AI 回复下方出现「插入当前笔记」动作。
- [ ] 插入后编辑器内容立即更新并触发自动保存。
- [ ] 若用户有选中段落，提供「替换选中」动作。
- [ ] 新增/更新单元测试覆盖 `useApplyAiOutput` 的新分支。

## 五、功能 3：时间轴筛选与折叠

### 5.1 需求描述

在时间轴视图顶部增加筛选栏，支持按以下维度快速过滤事件：

- 角色（多选）：只显示选中角色参与的事件。
- 地点（多选）：只显示选中地点相关的事件（事件表/地点表已有数据）。
- 状态（单选/多选）：draft / done / revise。
- 文本搜索：事件标题/描述关键词。

同时支持轨道折叠/展开，折叠后轨道标题保留为细条，事件隐藏，释放垂直空间。

### 5.2 影响文件

- 修改：`src/components/views/TimelineView.tsx`
- 新增：`src/features/timeline/useTimelineFilters.ts`
- 修改：`src/features/timeline/hooks.ts`（如需要共享 filter 状态）
- 修改：`src/i18n/locales/zh-CN.json` / `en.json`

### 5.3 实现要点

1. 新增 `useTimelineFilters` hook，管理 `filters` 状态（角色 ID 数组、地点 ID 数组、状态数组、搜索词、折叠轨道 ID 数组）。
2. 将 filters 状态持久化到 `localStorage`，key 为 `plotline-timeline-filters-{workspaceId}`。
3. 在 `TimelineView.tsx` 顶部工具栏下方新增筛选栏：
   - 角色下拉多选（使用 `useCharactersQuery`）。
   - 地点下拉多选（使用 `useLocationsQuery`）。
   - 状态下拉多选（使用现有状态枚举）。
   - 搜索输入框。
   - 清除筛选按钮。
   - 折叠/展开全部按钮。
4. 在事件渲染前应用 filters 计算可见事件集合。
5. 轨道渲染时根据折叠状态调整高度：折叠时高度约 24px，并显示事件数量徽标。
6. 右键菜单或轨道面板中增加「折叠轨道」/「展开轨道」入口。

### 5.4 验收标准

- [ ] 选择角色后，时间轴只显示该角色参与的事件，其他事件隐藏。
- [ ] 选择状态后，只显示对应状态事件。
- [ ] 搜索词匹配事件标题或描述时，只显示匹配事件。
- [ ] 折叠轨道后，该轨道事件隐藏，轨道标题保留并可展开。
- [ ] 刷新页面后筛选状态可恢复（持久化）。
- [ ] 新增/更新单元测试覆盖 filter 逻辑。

## 六、功能 4：模板向导空状态

### 6.1 需求描述

统一并强化各视图空状态，核心场景是「新建工作区时选择模板」。内置 3 种示例模板：

- **奇幻小说模板**：预置世界观、主要角色、地点、关键事件、三幕式大纲。
- **都市言情模板**：预置男女主角、关键地点、感情线事件、大纲。
- **跑团战役模板**：预置 NPC、场景地点、战役事件、章节大纲。

### 6.2 影响文件

- 修改：`src/components/ui/Feedback.tsx`（扩展 `EmptyState` 支持多个 action）
- 新增：`src/features/workspace/seedTemplates.ts`
- 修改：`src/features/workspace/api.ts`
- 修改：`src-tauri/src/commands/workspace.rs`
- 修改：`src-tauri/src/services/workspace.rs`
- 修改：`src-tauri/src/models/workspace.rs`（如 `CreateWorkspaceInput` 需增加 `templateId`）
- 修改：`src/components/views/WorkspaceSelector.tsx`
- 修改：各视图 `EmptyState` 调用点（如 TimelineView、CharactersView、OutlineView 等）
- 新增迁移：`src-tauri/migrations/XXX_workspace_templates.sql`（如需要存储自定义模板，v2.6.0 先使用硬编码内置模板）

### 6.3 实现要点

1. `seedTemplates.ts` 导出 `WORKSPACE_TEMPLATES` 数组，每项包含 `id`、`nameKey`、`descriptionKey`、`icon`、`data`：
   - `data` 包含轨道列表、事件列表、角色列表、地点列表、大纲节点列表。
2. 扩展 `EmptyState` 组件：支持 `actions?: React.ReactNode[]` 或 `action` 保持兼容。
3. 后端：
   - `CreateWorkspaceInput` 增加可选 `templateId?: string`。
   - `services/workspace.rs` 的 `create` 函数在 `templateId` 存在时，先创建基础工作区，再批量插入模板数据。
   - 所有插入在一个事务中完成。
4. 前端：
   - `WorkspaceSelector` 创建弹窗增加模板选择步骤或折叠面板。
   - 创建 API 调用传入 `templateId`。
5. 各视图空状态：当对应实体为空时，显示「从模板开始」或「添加第一个 XX」双 CTA。

### 6.4 验收标准

- [ ] 新建工作区弹窗可选择至少 3 种内置模板。
- [ ] 选择模板后创建的工作区包含示例角色、地点、事件、大纲。
- [ ] `EmptyState` 组件支持多个 action 且现有调用点无回归。
- [ ] 各主要视图（时间轴、角色、大纲、地图）空状态提供明确下一步操作。
- [ ] 后端新增/更新 Rust 单元测试覆盖模板创建工作区。

## 七、非功能项

| 项 | 说明 |
|---|---|
| 动画/前端质感 | 各新增面板、下拉菜单、chip 行均使用 `MOTION_BASE`（0.25s，ease `[0.16, 1, 0.3, 1]`）。筛选结果显示/隐藏使用 `AnimatePresence` 淡入淡出。 |
| 可访问性 | 所有新增按钮/输入框带 `aria-label`；下拉菜单使用 `role="menu"`、`role="menuitem"`。 |
| 测试 | 每个功能至少 1 个单元测试；E2E 覆盖「新建模板工作区 → 时间轴筛选 → AI 风格模板发送」。 |
| 类型 | 禁止 `any`/`@ts-ignore`；新增类型放 `src/types/` 并同步 Rust 模型。 |

## 十、风险与回退

| 风险 | 应对 |
|---|---|
| AI apply-to-doc 需要跨组件共享 editor 实例，可能引入耦合 | 使用独立 store 保存 editor 引用与选区，组件失焦时清空 |
| 时间轴筛选后性能下降 | 使用 `useMemo` 缓存可见事件；大量事件时考虑虚拟化 |
| 模板数据插入失败导致工作区半完成 | 所有模板插入在一个事务中，失败回滚 |
| Rust 本地测试环境仍被 Windows build-script panic 阻塞 | 依赖 CI 跑 `cargo test`；本地用 `pnpm` 前端测试兜底 |
