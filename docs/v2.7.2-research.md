## 竞品与时间轴工具

### 1.1 快速对比

| 工具 | 核心定位 | 信息密度 | 拖拽/边界 | 动画节奏 | 工具栏/标尺 | 事件卡片/连线 | 对 Plotline 的关键启示 |
|---|---|---|---|---|---|---|---|
| **Notion Timeline** | 数据库驱动的甘特视图 | 中高密度：一行一事件，按属性垂直分组 | 拖动条两端改起止，拖动整段平移；无复杂依赖线 | 150–200ms 淡入位移，长列表虚拟滚动 | 顶部标尺日/周/月/季/年，视图即筛选 | 横向条+文字标签，无连线 | AI 入口贴近上下文；视图即筛选；动画重性能 |
| **Aeon Timeline** | 专业叙事/项目时间轴 | 高密度：时间线、关系、地铁、叙述、大纲、思维导图多视图 | 直接拖动时间节点调时间与持续时长；自定义时间精度到秒 | 原生桌面拖拽，几乎无过渡动画 | 顶部时间标尺，支持自定义历法 | 事件卡片含角色/地点/属性；关系视图用连线展示事件-角色-地点交集 | 多视图是叙事刚需；叙述视图与时间轴解耦 |
| **TimelineJS** | 讲故事时间轴（展示向） | 低密度：单事件大图+媒体+长文本 | 基本不支持拖拽编辑 | 滑动切换，几乎无微动效 | 底部/顶部时间刻度 | 大卡片+媒体，无连线 | 导出叙事长图/PPT 可参考；但不适合密集编辑 |
| **Tiki-Toki** | 在线多媒体时间轴 | 中密度：3D/分类带视图，支持图片视频 | 网页端拖拽调整，边界感较弱 | 3D 切换较炫，可能过度 | 分类带（Category Band）视图有特色 | 故事（事件）按颜色分类，悬停展开 | 分类带/时间段着色可作为轨道衍生；3D 对创作工具价值有限 |
| **Obsidian Timeline** | Markdown 笔记生态插件 | 低密度：纯文本时间线 | 无拖拽，靠 Markdown 语法生成 | 纯 CSS 渲染，无动画 | 简单时间刻度 | 线条+节点，自定义 CSS | Markdown 原生友好；样式应基于 CSS 变量便于主题切换 |

### 1.2 Notion Timeline：视图即筛选与工具栏组织

- **布局与信息密度**：每个数据库条目渲染为横向甘特条，支持开始/结束时间属性；可按 Select / Person / Status 垂直分组，实现"视图即筛选"。[Notion 2.10: Timeline 终于来了 - 少数派](https://sspai.com/post/63572)
- **拖拽边界**：拖动条两端修改起止时间，拖动整段平移；没有复杂的依赖连线，保持简洁。
- **动画节奏**：条目移动以 150–200ms 淡入/位移为主，无弹性效果；长列表虚拟滚动；视图切换几乎即时。
- **工具栏组织**：顶部统一放置时间范围选择器、缩放控制、分组属性、筛选器，右侧展开属性面板。
- **对 Plotline 的启示**：
  1. 时间轴应强化"视图即筛选"——同一数据的不同透镜（时间轴/甘特/树/文本）共享筛选状态。
  2. 工具栏应按"时间范围 → 缩放 → 轨道/分组 → 筛选"从左到右组织，减少认知负荷。
  3. 事件卡片动画时长控制在 200ms 以内，优先保证拖拽跟手。

### 1.3 Aeon Timeline：多视图、自定义时间与拖拽边界

- **多视图与信息密度**：提供 Timeline（时间线）、Relationship（关系）、Subway（地铁）、Narrative（叙述）、Outline（大纲）、Mindmap（思维导图）、Spreadsheet（表格）七种视图。[Aeon Timeline 官网](https://www.timeline.app/)
- **自定义时间**：支持绝对时间、相对时间，精度到秒；可自定义一周天数、一天小时数，适合奇幻/科幻世界观。[退可搞定写作、进可任务管理，二合一时间轴式创作工具：Aeon Timeline - 少数派](https://sspai.com/post/76848)
- **拖拽边界**：在时间线视图中直接拖动时间节点调整时间位置和持续时间；从绝对切换到相对时间只需确定时间零点，时间轴直接变动。
- **叙述视图**：按幕/章节手动拖入事件，决定"故事讲述顺序"，与按时间排序的 Timeline 解耦，方便处理闪回、倒叙。
- **关系/地铁视图**：用连线或地铁图展示事件-角色-地点交集，筛选角色后清晰展示特定时间段的人物交集。
- **对 Plotline 的启示**：
  1. 当角色与地点增多时，关系/地铁视图能更清晰展示事件-角色-地点交集。
  2. 自定义历法/相对时间是奇幻/科幻创作的高阶刚需，可作为后续版本储备。
  3. 叙述编排视图（按幕/章节组织事件）是 v2.7+ 值得储备的方向。

### 1.4 TimelineJS & Tiki-Toki：展示向时间轴的卡片与标尺

- **TimelineJS**：由 Knight Lab 开发，适合新闻媒体讲故事。事件卡片默认含媒体、标题、日期、描述，纵向滑动切换。优势是多媒体嵌入和响应式，劣势是编辑能力弱。[Timeline JS 官网](https://timeline.knightlab.com/)
- **Tiki-Toki**：支持 3D 时间轴、分类带（Category Band）视图、时间段自定义背景图与颜色、多媒体嵌入。3D 视图对创作工具价值有限，但"分类带"和"时间段着色"可作为 Plotline 轨道/幕的衍生。[Tiki-Toki 官网](https://www.tiki-toki.com/)
- **对 Plotline 的启示**：
  1. 导出叙事长图/PPT 时可参考 TimelineJS 的卡片排版。
  2. 轨道可扩展为"分类带"或"幕带"，用背景色区分故事阶段。

### 1.5 Obsidian Timeline：Markdown 原生与可扩展性

- **使用方式**：通过 Markdown 标签 `timeline` / `timeline-labeled` 创建时间线，语法简单。
- **样式**：预定义 CSS 类控制线条、主体内容、活动颜色；支持自定义 CSS 变量。
- **对 Plotline 的启示**：
  1. AI 生成的内容可导出为 Markdown 时间线，方便用户迁移/分享。
  2. 时间轴样式应基于 CSS 变量（`src/styles/themes.css`），便于主题切换和自定义。

### 1.6 时间轴关键体验维度总结

| 维度 | 竞品最佳实践 | Plotline v2.7.2 可落地改进 |
|---|---|---|
| **信息密度** | 一行一事件，标题+时间+状态点默认显示，摘要/tag 默认折叠 | 事件卡片默认一行，hover 展开摘要与关联角色 |
| **拖拽边界** | Aeon 直接拖动节点调时间；Notion 拖动条两端/整段 | 拖拽时显示时间对齐辅助线，明确不能越界画布 |
| **动画节奏** | 150–200ms 淡入位移，无弹性 | 拖拽反馈 120ms，卡片入场 200ms，退出 120ms |
| **工具栏组织** | 时间范围 → 缩放 → 分组/轨道 → 筛选 → 视图切换 | 统一筛选栏按钮高度与垂直对齐，增加"跳转到今天" |
| **事件卡片视觉** | 轨道色条+表面色，hover 轻微上浮+阴影加深 | 强化轨道语义，选中态 ring+scale，hover 展开详情 |
| **标尺防重叠** | 日期标签自动抽稀/旋转，避免密集刻度重叠 | 标尺标签按缩放级别抽稀，"今天"标签边界保护 |
| **连线可视化** | 因果/伏笔用不同颜色曲线，hover 显示关系类型 | 因果实线、伏笔虚线，选中事件时相关连线高亮 |

## AI 助手界面惯例

### 2.1 五款 AI 助手对比

| 产品 | 能力说明入口 | `/` 命令菜单 | 上下文预算/可观测性 | RAG 检索反馈 | 快捷操作栏 | 对 Plotline 的启示 |
|---|---|---|---|---|---|---|
| **Claude** | 输入框下方 prompt 胶囊；Projects + Memories；Artifacts 结构化输出 | Claude Code `/` 命令；工具调用需审批 | 项目文件 + 历史对话 + 用户记忆 | 不明显 | prompt 胶囊 + Artifact 操作栏 | 记忆是长期创作核心；结构化输出应可编辑 |
| **Kimi** | 一键续写/润色/总结 chip | `@Kimi` 召唤 | 200万字超长上下文、联网搜索 | 不直接显示 | 输入框上方固定 chip | 中文用户爱"一键完成"；超长上下文是杀手锏 |
| **豆包** | 桌面端划词工具栏；多场景文本生成 chip | 输入框 `/` 快捷入口 | 多轮对话 + IP 化角色 | 不直接显示 | 划词悬浮条 + chip | 划词/选中即操作，减少打断创作流 |
| **通义千问** | 一键续写/润色/公文写作 chip | 支持 `@` 引用 | 1000万字文档解析、多文档总结 | 进度可视化 | chip + 文档解析卡片 | 进度可视化；提示词模板市场 |
| **GitHub Copilot Chat** | `/` slash 命令列表；Agent 模式入口 | `/test`、`/pr-review`、`/explain` | `@workspace`、`@terminal` 上下文引用 | 不明显 | slash 命令 + 内联补全 | 把高频操作固化为 `/` 命令 |

### 2.2 能力说明入口：从"聊天框"到"能力卡片"

- **Claude**：输入框下方提供 "Help me develop a unique voice" / "Improve my writing style" / "Brainstorm creative ideas" 等胶囊，点击直接带入提示词。[Claude 官网](https://www.anthropic.com/claude)
- **Kimi/豆包/通义**：输入框上方固定「续写 / 润色 / 总结 / 扩写 / 翻译」等一键按钮。
- **Copilot Chat**：`/test`、`/pr-review`、`/explain` 等 slash command 把 SOP 固化。[在 VSCode 中使用 GitHub Copilot Chat - CSDN](https://blog.csdn.net/wannianchuan/article/details/136035333)
- **对 Plotline 的启示**：
  1. AI 面板顶部应把能力做成可视化卡片/chip，而不是让用户在输入框里猜。
  2. 每个能力需配简短说明（hint）和适用上下文（如"需选中事件""需在当前时间轴视图"）。
  3. 引入 `/` 命令：在输入框输入 `/` 唤出能力菜单，支持键盘导航。

### 2.3 `/` 命令菜单的设计惯例

- **触发方式**：输入 `/` 或点击输入框旁的 sparkle/快捷图标。
- **菜单结构**：分组显示（创作/润色/分析/检查），每组 3–5 个命令；支持方向键选择、Enter 确认、Esc 关闭。
- **命令元信息**：图标 + 命令名 + 一句话描述 + 适用上下文提示。
- **对 Plotline 的启示**：
  1. 把高频创作 SOP 封装为 `/` 命令：如 `/续写` `/润色` `/检查时间线漏洞` `/生成角色小传` `/整理伏笔`。
  2. 命令应根据当前上下文动态可用/禁用（例如没有选中事件时 `/优化此事件` 禁用并提示原因）。

### 2.4 上下文预算与可观测性

- **Claude Projects**：可上传文件、设定长期上下文，Claude 跨对话记住项目记忆。
- **Kimi `@` 召唤**：用户可主动引用文档或上下文。
- **Copilot Chat `@workspace`**：明确告知 AI 基于项目索引回答。
- **对 Plotline 的启示**：
  1. AI 面板应展示当前上下文摘要：工作区名称、已选实体、启用的数据源、RAG 检索到的相关条目数量。
  2. 用户应能一键展开/收起上下文详情，看到"AI 参考了哪些设定"。
  3. 当上下文接近模型上限时，给出轻量提示（如"已引用 8 个角色、12 个事件，接近上下文上限"）。

### 2.5 RAG 检索反馈

- **现状**：多数 AI 助手不直接展示 RAG 检索过程，用户无法判断 AI"看到了什么"。
- **通义千问**：长文档解析时展示进度条与已解析段落。
- **对 Plotline 的启示**：
  1. AI 生成前显示"正在检索相关设定/角色/事件..."，并列出检索到的条目标题与相关度。
  2. 生成结果中如涉及特定设定，应可点击跳转到原文位置（类似引用链接）。
  3. 当 RAG 未检索到相关内容时，提示用户"未找到相关资料，AI 将基于通用知识回答"。

### 2.6 @mention / 工具调用：从对话到工作流

- **Kimi `@Kimi`**：基于当前上下文执行任务。
- **豆包桌面端划词工具栏**：选中文字后弹出悬浮条，提供翻译、润色、改写、生成大纲等操作。
- **Claude Code / Cowork**：Claude 可读取本地文件、调用工具并分步执行，用户审批后继续。
- **对 Plotline 的启示**：
  1. 在编辑器中选中文字后弹出 AI 工具条（润色、改语气、续写、插入到时间轴）。
  2. 在时间轴事件卡片右键菜单加入"AI 优化此事件""检查与此事件相关的逻辑冲突"。
  3. AI 生成的内容以 Artifact 卡片呈现，支持「插入到时间轴 / 插入到大纲 / 复制 / 弃用」。

## 番茄钟/MC 主题反馈

### 3.1 番茄钟竞品对比

| 产品 | 核心机制 | Gamification | 视觉/反馈 | 统计/目标 | 对 Plotline 的启示 |
|---|---|---|---|---|---|
| **Forest** | 种树：专注时树成长，离开应用树枯死 | 森林可视化、树种解锁、每日/每月挑战、真实植树 | 树从幼苗到茂盛的实时生长动画 | 按天/周/月统计、标签分类、连续打卡 streak | 正向可视化（成长）+ 负向约束（枯萎）结合最有效 |
| **Pomofocus** | 简洁在线番茄钟 | 无强游戏化，靠任务估算与报告 | 极简倒计时 + 任务列表 | 日/周/月视觉报告、项目追踪、CSV 导出 | 任务-番茄钟绑定、项目统计是刚需 |
| **Focus Plant** | 种植植物成长 | 植物成长、花园收集 | 卡通植物生长动画 | 基础统计 | 可作为 MC 主题外另一套"养成"皮肤 |

### 3.2 Forest：成长可视化与负向约束

- **核心循环**：选择专注时长与树种 → 树实时生长 → 完成加入森林；若切换应用则树枯死。[Forest 官网](https://www.forestapp.cc/)
- **Gamification 设计**：
  - 成长可视化：森林是专注时间的可视化记录。
  - 挑战与成就：每日/每月任务奖励专属树种。
  - 真实影响：用金币兑换真实植树（已超 200 万棵）。
  - 社交约束：Plant Together 房间，一人放弃全员树死。
- **对 Plotline 的启示**：
  1. MC 主题可引入"方块堆叠/植物生长"正向反馈，替代 Creeper 惊吓。
  2. 完成番茄钟后给予可收集的稀有方块/树种，形成个人"花园/矿场"。
  3. 连续打卡 streak 与每日目标结合，增强创作动力。

### 3.3 Pomofocus：任务绑定与统计

- **核心流程**：添加今日任务 → 为每个任务估算番茄数 → 选择任务开始专注 → 迭代直到完成。[Pomofocus 官网](https://pomofocus.io/)
- **统计功能**：日/周/月视觉报告，项目追踪，CSV 导出，Todoist 集成。
- **对 Plotline 的启示**：
  1. 番茄钟应与当前工作区/章节/角色绑定，形成"我在这个项目花了多少时间"统计。
  2. 增加任务估算与实际番茄数对比，帮助用户感知时间规划能力。

### 3.4 中文社区对 MC/像素主题的反馈

| 反馈 | 社区表达 | 可落地改进 |
|---|---|---|
| 像素字体热度 | "有人把我的世界的字体搬进了代码编辑器，GitHub 涨了一万多颗星" | 确保像素字体 12px/16px 清晰，仅用于标题/计时器 |
| 像素风治愈感 | "可爱、复古、氛围感" | 昼夜变化、方块堆叠进度等轻沉浸 |
| 可读性边界 | "像素字体在过小或非整数倍缩放下发虚" | 限制像素字体使用场景，编辑器正文不使用 |
| Creeper 惊吓 | "Creeper 弹窗吓人" | MC 彩蛋改为可关闭，增加频率限制 |
| UI 一致性 | "插图和色调都风格不统一" | 统一图标/配色设计系统，MC 纹理避免全局污染 |

## 可落地的前端动效建议

### 5.1 "炫技但不过度打扰"原则

| 原则 | 说明 | 反例 |
|---|---|---|
| **时长 120–250ms** | 微交互 120–150ms，视图切换 200–250ms | 超过 300ms 的入场动画 |
| **优先 transform/opacity** | 避免触发 layout/paint | width/height/margin 动画 |
| **ease-out 曲线** | `[0.16, 1, 0.3, 1]` 统一 | 弹性、弹跳、闪烁 |
| **可关闭/可降级** | 复杂动效应受 `prefers-reduced-motion` 与设置开关控制 | 强制播放的庆祝动画 |
| **服务核心任务** | 动画应强化用户当前操作，而非炫技 | 与当前任务无关的漂浮元素 |

### 5.2 可落地的微动效与实现思路

| 效果 | 适用场景 | 技术实现 | 性能注意 |
|---|---|---|---|
| **Shimmer 骨架屏** | 列表/卡片加载、AI 等待回复 | Tailwind `animate-pulse` 或 `background-position` 线性渐变动画 | 使用 `transform` 或 `background-position`，避免重排；受 `prefers-reduced-motion` 控制 |
| **Float 上浮** | 事件卡片 hover、按钮 hover | `translateY(-2px)` + `box-shadow` 加深 | 仅动画 `transform` 和 `opacity`，阴影变化会触发 paint，建议用伪元素或降低频率 |
| **Scale 缩放** | 选中态、能力 chip 激活、番茄钟完成 | `scale: 1.02` + `ring-accent ring-2` | 配合 `transform-origin`；避免大面积同时 scale |
| **卡片翻转** | 角色卡片正反面、事件详情摘要展开 | Framer Motion `rotateY` + `preserve-3d` | 开启 `backface-visibility: hidden`；减少同时翻转数量 |
| **粒子掉落** | 番茄钟完成、成就解锁 | Framer Motion + 少量绝对定位元素，或使用 Canvas | 粒子数控制在 20–50 个；结束后立即卸载 DOM |
| **连线流动** | 时间轴因果/伏笔连线高亮 | SVG `stroke-dashoffset` 动画或 GSAP DrawSVG | 仅对选中事件相关的连线动画，避免全局重绘 |
| **布局切换** | 视图切换、Tab 下划线滑动 | Framer Motion `layoutId` | 限制 `layout` 动画元素数量，避免全局重排 |
| **数字递增** | 统计视图数值、番茄钟累计时长 | Framer Motion `useMotionValue` + `useTransform` 或 GSAP `textContent` | 使用 `requestAnimationFrame` 或 Motion 内置动画，避免 setState 风暴 |

### 5.3 Framer Motion 实践

- **已使用**：`AnimatePresence`、`motion.div`、拖拽约束、`useMotionValue`。
- **问题**：`AnimatePresence mode="wait"` 在某些场景下会阻塞交互；退出动画时长与入场相同，导致"拖沓"感。
- **优化建议**：[Framer Motion & GSAP 实现酷炫动画 - 掘金](https://juejin.cn/post/7525395532066062345)
  - 入场 200ms，退出 120ms，符合"进入慢、离开快"的认知。
  - 对大量列表项使用 `layout` 动画时，加 `layoutId` 并限制数量，避免全局重排。
  - 使用 `useReducedMotion()` 统一降级。

### 5.4 GSAP 适合"炫技"时刻

- **文字拆分动画（SplitText）**：标题、空状态大标题可使用逐字/逐词淡入，但仅用于首屏或重要切换。
- **路径动画（MotionPath）**：时间轴连线可使用 GSAP 绘制 SVG 路径，hover 时高亮流动。
- **ScrollTrigger**：叙事长图导出预览时可使用，但在主界面避免过度使用。

### 5.5 CSS Houdini 与 View Transitions API

- **View Transitions API**：适合视图切换（如时间轴 ↔ 大纲 ↔ 角色），可实现"共享元素过渡"。[View Transitions API 实用教程 - 掘金](https://juejin.cn/post/7289952867053731899)
  - 但 WebView2 支持度需验证，建议作为渐进增强。
- **CSS Houdini**：可用于自定义背景纹理（如 MC 主题方块、纸张纹理），但不应阻塞主线程。[CSS Houdini:一种足以颠覆想象的 API - SegmentFault](https://segmentfault.com/a/1190000046008059)

### 5.6 性能注意事项

1. **优先使用 `transform` 和 `opacity`**：避免动画 `width`、`height`、`top`、`left`、`margin` 等会触发 layout 的属性。
2. **谨慎使用 `will-change`**：只在动画即将发生时添加，动画结束后移除，避免过量创建合成层。
3. **限制同时动画元素数量**：列表 stagger delay 不宜过小，避免同时触发大量动画。
4. **尊重 `prefers-reduced-motion`**：对 shimmer、粒子、昼夜渐变等循环/复杂动画提供降级。
5. **避免在滚动时触发动画**：使用 `IntersectionObserver` 或 Framer Motion `whileInView` 控制触发动画。
