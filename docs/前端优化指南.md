## 组件组织约定

1. **一个文件一个主要组件**。工具函数 / 多个紧密相关的纯函数可放同一文件。
2. **组件导出**：当前仓库以命名导出为主（`export function EventCard`），新组件保持与所在目录风格一致即可。
3. **import 顺序**：
   ```tsx
   import { useState } from 'react';
   import { motion } from 'framer-motion';
   import { Button } from '@/components/ui/Button';
   import { useUIStore } from '@/stores/ui';
   import type { Event } from '@/types/event';
   ```
4. **Props 类型声明**放在组件实现之前：
   ```tsx
   interface EventCardProps {
     event: Event;
     onSelect?: (id: string) => void;
   }
   ```
5. **子组件**如需复用范围小，直接写在同一文件主组件之后；若被多个文件引用，提升到 `src/components/ui/` 或 `src/features/<domain>/components/`。
6. **禁止**组件内直接调用 Tauri IPC，必须通过 `src/features/<domain>/api.ts` 封装。

## 动效系统

核心文件：`src/lib/motion.ts`、`src/hooks/useAmbientAnimation.ts`、`src/stores/motion.ts`。

### 现有 Token

```ts
export const EASE_STANDARD = [0.16, 1, 0.3, 1] as const;

export const MOTION_FAST: Transition = { duration: 0.12, ease: EASE_STANDARD };
export const MOTION_BASE: Transition = { duration: 0.18, ease: EASE_STANDARD };
export const MOTION_SLOW: Transition = { duration: 0.20, ease: EASE_STANDARD };
export const MOTION_DRAG_RETURN: Transition = { duration: 0.20, ease: EASE_STANDARD };
```

| Token | 时长 | 用途 |
|-------|------|------|
| `MOTION_FAST` | 120ms | 按钮 tap、Dialog overlay、微交互 |
| `MOTION_BASE` | 180ms | 视图切换、Toast、Sidebar、卡片入场 |
| `MOTION_SLOW` | 200ms | Splash 退场、骨架屏淡入、需要节奏舒缓的场景 |
| `MOTION_DRAG_RETURN` | 200ms | 时间轴卡片拖拽释放后的归位动画 |

### v2.8.0 Task 2 规划新增

- `MOTION_SPRING`：基于弹簧物理的过渡，用于需要“苹果级顺滑”的视图切换。
- `MOTION_LAYOUT`：专为 Framer Motion 的 `layout` / `layoutId` 动画优化的过渡，减少重排抖动。

### 缓动与时长约定

- 统一使用 `EASE_STANDARD = [0.16, 1, 0.3, 1]`（ease-out-expo 风格）。
- 核心交互控制在 **120–200ms**；Splash、骨架屏等可放宽到 250–300ms。
- 避免弹性、弹跳、闪烁、旋转 loading；加载态使用骨架屏 + `pulse-soft`。

### `prefers-reduced-motion` 处理

```ts
const reduced = useReducedMotion();
const { transition, animate } = useAmbientAnimation();
```

- `useAmbientAnimation()` 会综合 `motionStore.animationsEnabled`、`uiStore.enhancedAnimations` 与系统 `prefers-reduced-motion`。
- 当任一来源要求减少动效时，返回 `transition: { duration: 0 }`、`animate: false`。
- `src/styles/themes.css` 通过 `--motion-enabled` 变量统一关闭 CSS 动画：
  ```css
  @media (prefers-reduced-motion: reduce) {
    :root { --motion-enabled: 0; }
    * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }
  }
  ```

## 状态管理

### Zustand Store

位于 `src/stores/`，关键 store：

| Store | 职责 |
|-------|------|
| `ui.ts` | 侧边栏折叠、AI 面板开关、`enhancedAnimations`、主题/字体/强调色应用 |
| `motion.ts` | 全局动画开关 `animationsEnabled`、花俏动画 `fancyAnimationsEnabled` |
| `pomodoro.ts` | 番茄钟状态、成就、阶段、完成动画 |
| `aiContext.ts` | AI 助手上下文源选择 |
| `historyStore.ts` | Undo/Redo 栈 |
| `editorSelection.ts` | 编辑器选区共享 |
| `toast.ts` | 全局 Toast 通知 |

### TanStack Query

用于所有服务端数据（工作区、事件、轨道、角色、设置等）：

```ts
// src/features/timeline/hooks.ts
export function useTimelineEventsQuery(workspaceId: string) {
  return useQuery({
    queryKey: ['events', workspaceId],
    queryFn: () => listEvents(workspaceId),
  });
}
```

### 何时用 useState / useReducer

- 组件内部 UI 状态（如弹窗开关、临时表单值、本地 hover 态）。
- 不需要跨组件共享的复杂局部状态可用 `useReducer`。
- 跨组件共享或需要持久化的状态再使用 Zustand。

## 样式规范

1. **Tailwind 类名优先**。所有样式通过 Tailwind 类名实现。
2. **禁止裸 CSS**。组件级私有样式使用 Tailwind 任意值 / 任意选择器语法：
   ```tsx
   <div className="rounded-[8px] shadow-[var(--shadow-card)] [&_[data-slot]]:p-0" />
   ```
3. **禁止内联 `style={{}}` 写颜色/字体**。仅允许动态计算值（如时间轴卡片的 `left`、`width`、轨道色）。
4. **主题颜色**使用 CSS 变量对应的 Tailwind 别名：`bg-bg-base`、`text-text-primary`、`border-border`、`bg-accent`。
5. **响应式断点**使用 Tailwind 默认：`sm:`、`md:`、`lg:`、`xl:`。
6. **圆角/阴影**：卡片 `8px`、按钮 `6px`、对话框 `12px`；阴影使用柔和弥散的 `--shadow-card-soft` / `--shadow-elevated-soft`。

## IPC 与数据流

### 调用路径

```text
组件/视图
  -> src/features/<domain>/hooks.ts（TanStack Query）
    -> src/features/<domain>/api.ts（封装 invoke）
      -> src/lib/ipc.ts（判断 Tauri / Web 模式）
        -> Tauri Rust 命令 / mockIpc（纯 Web 模式）
```

### 示例

```ts
// src/features/timeline/api.ts
import { invoke } from '@/lib/ipc';
import type { Event } from '@/types';

export function listEvents(workspaceId: string): Promise<Event[]> {
  return invoke<Event[]>('list_events', { workspaceId });
}
```

组件中：

```tsx
import { useTimelineEventsQuery } from '@/features/timeline/hooks';

function TimelineView({ workspaceId }: { workspaceId: string }) {
  const { data: events = [] } = useTimelineEventsQuery(workspaceId);
  // ...
}
```

### 禁忌

- **禁止**在组件中直接 `import { invoke } from '@tauri-apps/api/core'`。
- **禁止**在组件中直接拼接 SQL 或调用后端原始命令。
- 错误处理统一在 Query 的 `onError` 或 `useToast` 中，禁止 `try/catch` 后 `console.log` 吞掉。

## 时间轴坐标与动画编排最佳实践（v3.1.0+）

### 1. 单一坐标源

所有时间↔像素换算必须通过 `viewportState` + `getXAtTime` / `getTimeAtX` 纯函数（`src/features/timeline/timelineGrid.ts`）完成，**禁止**组件内独立实现 `timeToX` / `xToTime`。

- **背景**：v3.1.0 前标尺、事件卡片、连接线、Today 线各自维护坐标换算逻辑，缩放/平移/滚动后出现标尺与卡片错位、连接线偏离锚点等问题。
- **正确做法**：
  ```ts
  import { getXAtTime, getTimeAtX, type ViewportState } from '@/features/timeline/timelineGrid';

  const x = getXAtTime(event.dateTime, viewportState);
  const time = getTimeAtX(pointerX, viewportState);
  ```
- **禁止做法**：
  ```ts
  // ❌ 禁止：组件内独立实现 timeToX
  const x = (event.dateTime - startTime) * pixelsPerDay;
  ```
- **覆盖范围**：标尺刻度定位、事件卡片 `left` / `width`、连接线起点/终点、Today 参考线、拖拽时指针→时间换算，全部走单一源。
- **测试保障**：`timelineGrid.test.ts` 中 `getXAtTime` / `getTimeAtX` 在年/月/日/时四档 zoom 下的精度测试（误差 ≤ 1px）。

### 2. 动画编排层

跨视图/批量元素的动画必须通过 `src/lib/motionOrchestrator.ts` 的 `getScenePreset(scene, { enhanced })` 消费场景预设，统一管理 stagger / delay / exit timing。

- **背景**：v3.1.0 前视图切换、卡片批量入场、拖拽归位 + 连接线、AI 面板展开、Sidebar 导航入场等动画分散在各组件，缓动曲线与 stagger 计算不一致，且 `prefers-reduced-motion` 处理重复。
- **5 个场景预设**：
  | Scene | 用途 | 关键参数 |
  |-------|------|----------|
  | `viewSwitch` | 路由视图切换 | 200ms 淡入 + 轻微 y 位移 |
  | `cardBatchEnter` | 事件卡片批量入场 | stagger 30ms，最多 6 张 |
  | `dragReturnWithConnections` | 拖拽归位 + 连接线 | 200ms spring + 连接线同步 |
  | `aiPanelExpand` | AI 面板展开 | 250ms 宽度/透明度 |
  | `sidebarNavEnter` | Sidebar 导航入场 | stagger 40ms |
- **正确用法**：
  ```ts
  import { getScenePreset } from '@/lib/motionOrchestrator';
  const preset = getScenePreset('cardBatchEnter', { enhanced });
  // preset.container / preset.item / preset.staggerChildren
  ```
- **退化策略**：`prefers-reduced-motion` 或"增强动效"开关关闭时，所有场景统一退化为 200ms 同步淡入，不再触发 stagger。
- **禁止做法**：在组件内手写 `staggerChildren` / `delayChildren` / `exit` 组合，绕过编排层。

## 已知问题与优化方向

1. **时间轴长列表渲染性能（已缓解）**
   - `TimelineView.tsx` 已在 `TrackLane` 内实现基于滚动位置的可视区裁剪，仅渲染缓冲区范围内的事件卡片与连线。
   - 若事件数量超过 200 条仍出现掉帧，可进一步引入 `@tanstack/react-virtual` 对轨道列表和事件卡片做完整虚拟化。

2. **Framer Motion `layout` prop 使用（已规范）**
   - `EventCard` 已按条件启用 `layout`（仅在 `enhancedAnimations` 开启且非拖拽状态），显著降低静态事件的重排开销。
   - 后续新增组件应继续遵守“只有位置/尺寸会变化且需要平滑过渡的元素才开启 `layout`”的规则。

3. **视图切换动画仍有优化空间**
   - `WorkspaceLayout` 与 `AnimatedRoutes` 已使用 `MOTION_FAST` / `MOTION_BASE`。
   - Task 2 计划引入 spring 过渡与更精细的 GPU 加速，进一步消除快速切换时的闪烁与重排。

4. **设置页可进一步分组/搜索**
   - 当前设置项较多，分类与说明文案不足。
   - Task 5 计划按“外观/编辑器/AI/数据/关于”分组，并增加搜索框与即时预览。

5. **空状态文案与引导可继续打磨**
   - 部分视图空状态仅有静态文案，缺少引导 CTA。
   - Task 6 计划统一空状态插画、CTA 按钮，并增加首次进入工作区的分步引导。

### Task 3：时间轴拖拽体验升级

- 为事件卡片添加拖拽 ghost / preview 指示器。
- 优化 `handleEventDragEnd`，使用 layout 动画平滑归位。
- 使用 `React.memo` / `useMemo` 减少拖拽过程重渲染。
- 修复跨轨道误触与拖拽边界问题。
- 更新 `tests/e2e/timeline-drag-boundary.spec.ts` 覆盖新行为。

### Task 4：TimelineView 性能优化（已完成）

- **减少不必要重渲染**
  - 将 `EventCard`、`TrackLane`、`DateRuler`、`ConnectionLayer` 提取为 `React.memo` 组件，props 保持引用稳定。
  - 使用 `useMemo` 缓存：`visibleTracks`、`filteredEvents`、`timeScale`、`totalWidth`、`eventLayout`、`timelineMinHeight`、`eventPositions`、`eventsByTrack`、日期标尺刻度、`dragConstraints`、轨道事件坐标等。
  - 使用 `useCallback` 缓存：`handleEventDragEnd`、`handleSaveEvent`、`handleDeleteEvent`、`handleAddEvent`、`handleCanvasDoubleClick`、`cycleZoom`、`handleSelectEvent` 等事件处理函数。
- **Framer Motion 使用优化**
  - `EventCard` 仅在 `enhancedAnimations` 开启且非拖拽状态下启用 `layout`，避免大量静态事件同时参与布局动画。
  - 日期标尺、轨道背景、添加按钮等静态元素不再使用 `motion` 组件。
  - 移除未使用的 `MOTION_DRAG_RETURN` 导入，统一使用 `MOTION_FAST` / `MOTION_BASE` / `MOTION_LAYOUT`。
- **长列表 / 可视区裁剪**
  - 在 `TrackLane` 内实现基于画布滚动位置的事件可见性裁剪：只渲染 `scrollLeft - buffer` 到 `scrollLeft + viewportWidth + buffer` 范围内的事件。
  - `ConnectionLayer` 同样按可见区裁剪连线，只绘制进入缓冲区的连线。
  - 缓冲距离为 `EVENT_CARD_MAX_WIDTH * 2`，在防止裁剪闪烁与减少渲染量之间取得平衡。
- **状态与 effect 清理**
  - 合并/精简 `useEffect`，避免顶层 state 频繁变化触发整棵树渲染。
  - 将拖拽状态 `draggingEvent` 下沉到事件卡片层，减少父组件重渲染。
