# v2.6.1 社区与竞品调研：时间轴 / AI 状态 / 字体 / 动画

> 调研目标：为修复时间轴筛选下拉定位、「Today」标记重叠、事件卡片内部布局、设置页视觉层级、AI 设置状态可见性提供可落地的设计参考。
> 来源：B站/哔哩哔哩、V2EX、知乎、NGA、CSDN、掘金及海外竞品（Aeon Timeline、Plottr、Milanote、Notion、Claude Desktop、ChatGPT Desktop 等）。
> 日期：2026-06-27

## 1. 时间轴可视化布局模式与常见痛点

### 1.1 布局模式

| 模式 | 典型产品/社区案例 | 适用场景 | 对 Plotline 的启示 |
|---|---|---|---|
| **甘特式横向轨道（Gantt-like）** | 51mazi 事序图、Aeon Timeline、Unity Timeline | 多角色/多地点并行事件，强调时间跨度与因果关系 | Plotline 的「轨道」设计与此一致，但需解决轨道多时垂直空间爆炸问题 |
| **垂直时间线（Vertical Timeline）** | TimelineJS、Cool Timeline、知乎/头条时间轴 | 单线叙事、阅读流强 | 不适合小说多线并行，但可作为导出/预览视图参考 |
| **棋盘/泳道（Swimlane）** | Plottr、Milanote Board | 按角色/地点分组，卡片自由排布 | 折叠轨道、筛选后保留标题条的思路与泳道折叠一致 |
| **树状/层级时间轴** | Heptabase Timeline、Notion Timeline | 事件嵌入大纲/卡片体系 | 可将大纲节点与时间轴事件双向关联，减少重复录入 |

### 1.2 社区高频痛点（B站/知乎/CSDN/掘金）

1. **轨道过多导致纵向滚动过长**
   - 参考：掘金《小说写作中的时间轴管理：基于 Vue 3 的事序图技术实现》提到「轨道折叠」与「按角色/地点筛选」是降低视觉复杂度的核心手段。
   - 落地：v2.6.0 已引入轨道折叠，但折叠后标题条应保留拖拽排序入口与当前筛选命中数。

2. **筛选条件与视图状态不同步**
   - 参考：CSDN《小说创作中的时间轴体验设计：事序图交互与用户体验优化》指出「筛选后空状态必须给出明确反馈，否则用户会怀疑数据丢失」。
   - 落地：筛选结果数为 0 时，应在时间轴中央显示「无匹配事件」并提示「清除筛选条件」。

3. **事件卡片信息密度过高**
   - 参考：B站/知乎多位网文作者提到「写小说时间轴最怕一行字塞满角色、地点、状态，看得眼累」。
   - 落地：卡片主视觉应只保留「标题 + 时间」，角色/地点/状态用彩色小 tag 在底部一行收起，hover 再展开详情。

4. **「Today / 当前时间」标记与其他元素重叠**
   - 参考：Notion Calendar、Google Calendar 均采用「细线 + 悬浮胶囊标签」方案，标签默认吸附在可见区域顶部，随滚动保持可见。
   - 落地：Plotline 的 Today 标记可改为「垂直细线 + 顶部吸顶标签」，避免标签被事件卡片遮挡。

## 2. AI 助手连接状态可见性与配置 UX 模式

### 2.1 竞品模式

| 产品 | 状态展示位置 | 配置入口 | 失败反馈 |
|---|---|---|---|
| **Claude Desktop** | 对话窗口顶部模型选择器旁 | Settings → Developer → API Key | 仅配置阶段报错，对话中无持续状态灯 |
| **ChatGPT Desktop** | 右上角模型下拉（无状态灯） | Settings → Personalization / Data controls | 网络失败时以 toast 提示 |
| **NoteGen (V2EX/Tauri)** | 侧边栏底部状态条 + 设置页 | Settings → AI Provider | 连接测试按钮 + 错误码提示 |
| **Cursor / Windsurf** | 底部状态栏连接指示灯 | Settings → Models / API Key | 状态栏红/黄/绿点 + hover 显示模型名与延迟 |

### 2.2 社区反馈（V2EX/掘金/CSDN）

1. **状态必须「一眼可见」**
   - V2EX 讨论中多次提到「AI 笔记类应用如果 AI 没连上，用户输入半天才发现，体验极差」。
   - 建议：在 AI 面板输入框上方常驻一行状态条：绿点「已连接 · 模型名」、黄点「配置中」、红点「连接失败 · 点击重试」。

2. **配置过程需要「测试连接」反馈**
   - 参考：NoteGen 与众多 Tauri AI 工具都在设置页放置「验证」按钮，避免用户保存后才发现 key 无效。
   - 建议：AI 设置页增加「测试连接」按钮，返回「成功/失败 + 错误码 + 常见原因」。

3. **模型选择器应显示可用模型**
   - 参考：Claude Desktop 的模型下拉仅显示固定列表；社区更倾向 Cursor 的「模型名 + 当前 key 可访问」的过滤方式。
   - 建议：根据当前 provider 与 key 拉取可用模型列表，不可用的模型置灰并提示原因。

4. **错误信息要具体、可解决**
   - 参考：掘金《Claude Desktop 实战入门指南》中把「Missing Qt platform plugin」等报错与解决方案并列展示。
   - 建议：AI 设置错误不应只显示英文报错，应映射为中文：「API Key 无效，请检查密钥是否复制完整」「网络超时，请检查代理设置」。

## 3. 字体选择/预览 UI 模式（像素字体与 得意黑/Smiley Sans）

### 3.1 字体选择器通用模式

| 模式 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| **下拉列表 + 预览短语** | 节省空间、选择高效 | 预览面积小 | 设置页字体快速切换 |
| **卡片网格 + 实时大标题预览** | 视觉冲击强、风格对比明显 | 占用空间大 | 字体主题市场/首次配置向导 |
| **列表 + 右侧预览区** | 信息完整、可滚动对比 | 布局较重 | 专业排版软件 |
| **搜索/分类筛选** | 字体多时必须 | 实现成本略高 | 本地字体库/系统字体选择 |

### 3.2 像素字体与 得意黑 的社区用法

1. **像素字体**
   - 社区热门：Zpix、方舟像素字体（Ark Pixel Font）、Fusion Pixel Font。
   - 特点：8/10/12/16px 多尺寸，适合复古/游戏/番茄钟 MC 主题。
   - 选择器要点：必须显示「尺寸标签」，因为像素字体在不适配的尺寸下会发虚。
   - 参考：B站/CSDN 多篇像素字体指南强调「预览文案要包含中文、英文、数字和常用标点」。

2. **得意黑 / Smiley Sans**
   - 由 B站 UP 主 oooooohmygosh 与 atelierAnchor 锚坞发布，开源免费商用。
   - 特点：窄斜体、几何感强、弯钩造型，适合标题/强调，不适合大段正文。
   - 选择器要点：预览区应区分「标题字号」与「正文字号」，避免用户误用作编辑器字体。

### 3.3 对 Plotline 的建议

- 字体选择器统一预览文案：`「 plotline  Plotline  12345  ，。！？」`（含大小写、中英文、数字、标点）。
- 将字体分为「界面字体 / 编辑器字体 / 像素主题字体」三类，每类单独选择。
- 对像素字体显示推荐尺寸徽章（如 12px / 16px）。
- 得意黑 仅放入「界面字体」或「标题字体」候选，不作为默认编辑器字体。

## 4. 桌面创意 App 的平滑动画过渡

### 4.1 通用原则

- 时长：200–300ms，曲线 `cubic-bezier(0.16, 1, 0.3, 1)`（与 Plotline 现有规范一致）。
- 避免：弹性、旋转 loading、闪烁。
- 参考：B站/掘金 Framer Motion 与 GSAP 教程中反复强调「桌面端动画要比移动端更克制，避免打扰长文本阅读」。

### 4.2 具体场景模式

| 场景 | 推荐实现 | 注意事项 |
|---|---|---|
| **视图切换（Timeline ↔ Outline ↔ Notebook）** | `AnimatePresence` + `layoutId` 共享元素过渡 | 避免整个页面 fade 导致闪白；可让侧边栏保持固定，仅内容区滑动 |
| **轨道折叠/展开** | `framer-motion` 的 `animate={{ height: 'auto' }}` | 配合 `layout` prop，让下方轨道平滑补位，不要突兀跳动 |
| **筛选条件出现/消失** | 筛选 chip 从上方滑入，带 stagger 0.05s | 让用户感知到「筛选在变化」 |
| **事件卡片 hover/selected** | scale 1.02 + 阴影提升 + border 颜色过渡 | 不要位移，避免布局抖动 |
| **AI 回复流式输出** | 打字机效果 + 光标脉冲 | 底部滚动应平滑，不要跳到新消息顶部 |
| **设置页 tab 切换** | 内容区 opacity + translateX(8px) 轻量过渡 | 保持左侧导航稳定，符合用户心智模型 |

### 4.3 性能提示

- 长列表使用 `will-change: transform` 或 `content-visibility`。
- 时间轴滚动时暂停非可见区域动画，避免掉帧。
- 参考：掘金 GSAP 教程提到「多轨道时间轴动画应使用 `gsap.timeline()` 统一编排，避免多个独立动画抢占主线程」。

## 5. 针对 Plotline v2.6.1 的具体修复建议

### 5.1 时间轴筛选下拉定位

- **问题**：下拉菜单可能被时间轴边缘或窗口底部截断。
- **方案**：使用带碰撞检测的下拉组件（如 Radix UI DropdownMenu 的 `collisionPadding` + `sideOffset` + `avoidCollisions`）。
- **落地**：在筛选按钮下方 4px 弹出，若下方空间不足则自动翻转至上方的，同时限制最大高度并内部滚动。

### 5.2 「Today」标记标签重叠

- **问题**：Today 竖线标签与事件卡片/轨道标题重叠。
- **方案**：
  1. 竖线变细（1px → 2px 高对比 accent 色）。
  2. 标签改为顶部吸顶胶囊，始终位于时间轴可视区域顶部，不与卡片同层。
  3. 标签背景加半透明蒙层，避免与网格线混淆。

### 5.3 事件卡片内部布局

- **问题**：角色/地点/状态信息堆叠，视觉拥挤。
- **方案**：
  1. 卡片分三层：标题层（最大字号）、摘要层（1 行截断）、元信息层（tag 行）。
  2. tag 行只显示前 3 个，超出用「+N」收起，hover 卡片时展开。
  3. 状态用左侧 3px 色条区分，减少一个 tag 占用。

### 5.4 设置页视觉层级

- **问题**：设置项平铺，难以扫描。
- **方案**：
  1. 左侧导航 + 右侧卡片分组的经典双栏布局。
  2. 每组设置用卡片容器，标题 16px 加粗 + 描述文字 13px 次要色。
  3. AI 设置组置顶「连接状态卡片」，显示 provider / 模型 / 状态灯 / 测试按钮。
  4. 危险操作（清空数据、重置）放在最底部，按钮用破坏性颜色。

### 5.5 AI 设置状态可见性

- **问题**：用户难以判断 AI 是否可用。
- **方案**：
  1. **全局状态条**：AI 面板顶部常驻一行，显示绿/黄/红点 + 文字。
  2. **设置页状态卡片**：大号状态灯 + 模型名 + 剩余额度/延迟（如 provider 支持）。
  3. **测试连接按钮**：点击后显示 loading → 成功/失败，失败时展开折叠的排查清单。
  4. **失败兜底**：连接失败时输入框自动禁用并提示「AI 服务未配置」，避免用户输入后无响应。

## 7. 关键结论

1. **时间轴**：多轨道 + 折叠 + 筛选是行业共识；Today 标记应采用「吸顶胶囊标签」避免重叠；事件卡片需分层、tag 收起、状态用色条。
2. **AI 状态**：状态条应常驻且直观，设置页需要「测试连接」与具体中文错误提示，失败时禁用输入框。
3. **字体选择器**：统一预览文案、分类选择、像素字体标注尺寸、得意黑仅作标题/界面字体。
4. **动画过渡**：保持 200–300ms ease-out，视图切换用共享元素/轻量位移，轨道折叠用高度动画补位。
5. **设置页**：双栏布局 + 卡片分组 + 顶部 AI 状态卡片 + 危险操作置底。
