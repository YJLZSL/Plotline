# v2.7.1 打磨升级调研报告

> 调研日期：2026-06-27  
> 目标版本：Plotline v2.7.1（v2.7.0 体验问题专项修复）  
> 调研范围：竞品时间轴工具、AI 创作前端界面、中文社区反馈、动画/前端效果  
> 输出目标：把 v2.7.0 用户反馈的"Markdown 星号仍显示、动画拖沓、时间轴问题、AI 能力不明确、番茄钟重置成就失败、MC 元素体验差、缺乏炫技效果、主题 UI/字体兼容"等问题，转化为可直接落地的功能点与文件模块。

## 一、背景与目标

### 1.1 v2.7.0 实际体验反馈

在 v2.7.0 迭代后，内部/种子用户反馈的核心问题可归纳为 8 类：

| 问题类别 | 具体表现 | 对 v2.7.1 的诉求 |
|---|---|---|
| **Markdown 渲染** | AI 回复中的 `**粗体**`、`*斜体*` 仍被原样渲染为文本 | 任何情况下 Markdown 标记都能正确解析 |
| **动画拖沓** | AI 面板滑入、气泡入场、视图切换感觉"慢半拍" | 快而稳的反馈，减少 layout 动画与 AnimatePresence 阻塞 |
| **时间轴 UI** | 轨道留白多、事件卡片拥挤、"今天"标签遮挡、缩放后定位困难 | 提升信息密度与视觉关联，强化标尺与锚点 |
| **时间轴交互/逻辑** | 拖拽边界不清、事件与轨道关系弱、筛选后定位丢失、连线关系难读 | 明确拖拽约束、增强轨道语义、优化筛选-定位联动 |
| **AI 能力不明确** | 快捷 chip 多但不知道各自能干什么、上下文范围不可见、无 @mention/工具调用 | 把能力卡片化、上下文可解释、引入 slash/@ 快捷入口 |
| **番茄钟重置成就** | 重置成就按钮无反馈、成就状态与 UI 不同步 | 明确成功反馈、修复状态同步 |
| **MC 元素体验差** | 像素字体糊、Creeper 弹窗吓人、昼夜/方块反馈单一 | 限制像素字体使用场景、MC 彩蛋可关闭、增加正向反馈 |
| **缺乏炫技效果** | 整体偏素，缺少让人"哇"一下的瞬间 | 在关键节点加入克制但令人印象深刻的微动效 |
| **主题 UI/字体兼容** | 得意黑/像素字体回退不优雅、主题切换后部分组件未跟随 | 统一字体回退栈、主题变量全覆盖 |

### 1.2 调研目标

1. **竞品时间轴工具**：看 Notion Timeline、Aeon Timeline、Figma 社区模板、Obsidian Timeline 如何解决"信息密度、缩放、拖拽、筛选、多视图"问题。
2. **AI 创作前端界面**：看 Claude、Kimi、豆包、通义千问、GitHub Copilot Chat 如何让用户一眼看懂 AI 能做什么，以及如何处理 Markdown 渲染。
3. **中文社区反馈**：汇总 B 站、知乎、V2EX、NGA、小红书对叙事工具/写作软件/时间轴工具的 UI、动画、AI、MC/像素风、番茄钟的真实声音。
4. **动画与前端效果**：提炼 Framer Motion、GSAP、CSS Houdini、View Transitions API 的"炫技但不过度打扰"实践。

## 二、竞品分析

### 2.1 快速对比

| 工具 | 核心定位 | 时间轴布局 | AI 入口 | 动画风格 | 对 Plotline 的关键启示 |
|---|---|---|---|---|---|
| **Notion Timeline** | 数据库驱动的甘特视图 | 横向时间条，按属性垂直分组 | 行内 Ask AI、`/ai` 斜杠命令、Notion Agent | 极简、150–200ms 淡入位移，长列表虚拟滚动 | AI 入口应贴近上下文；视图即筛选；动画重性能 |
| **Aeon Timeline** | 专业叙事/项目时间轴 | Timeline / 关系 / 地铁 / 叙述 / 大纲 / 思维导图 | 无原生 AI | 原生桌面拖拽，几乎无过渡动画 | 多视图切换是叙事刚需；叙述视图与时间轴解耦 |
| **Figma 社区模板** | 设计资产与交互原型 | 时间轴模板多为"轨道+节点卡片+连线"的横向/纵向布局 | 无 | 依赖模板作者，常见 hover scale、卡片展开 | 卡片分层、hover 展开详情、连线的视觉引导值得借鉴 |
| **Obsidian Timeline** | Markdown 笔记生态插件 | 通过 `timeline` 标签生成纵向/横向时间线 | 可接入本地 AI 插件 | 纯 CSS 渲染，无动画 | Markdown 原生友好、可自定义 CSS 变量、轻量可扩展 |

### 2.2 Notion Timeline：AI 入口与视图即筛选

- **布局**：每个数据库条目是横向甘特条，支持开始/结束时间；按 Select / Person / Status 垂直分组；标尺支持日/周/月/季/年缩放。
- **交互**：拖动条两端修改起止时间，拖动整段平移；无复杂依赖线，保持简洁。
- **AI 入口**：
  - 行内「Ask AI」：在任意块末尾按空格或点击 sparkle 图标唤出。
  - `/ai` 斜杠命令：直接唤起 AI 菜单。
  - Notion Agent：2026 年主推 Custom Agents，可在后台自动执行任务。
- **动画**：条目移动以 150–200ms 淡入/位移为主，无弹性效果；长列表虚拟滚动；视图切换几乎即时。

**对 Plotline 的启示**：
1. AI 入口应贴近上下文——事件卡片、大纲节点、编辑器行内都能一键唤起。
2. 把常用创作 SOP（如"生成角色小传""检查时间线漏洞"）封装为 `/` 命令或 `@` 召唤。
3. Timeline 应强化"视图即筛选"——同一数据的不同透镜（时间轴/甘特/树/文本）共享筛选状态。
4. 时间轴动画时长控制在 200ms 以内，优先保证拖拽跟手。

### 2.3 Aeon Timeline：多视图是叙事刚需

- **多视图**：Timeline（时间线）、Relationship（关系）、Subway（地铁图）、Narrative（叙述）、Outline（大纲）、Mindmap（思维导图）、Spreadsheet（表格）。
- **自定义时间**：支持绝对时间、相对时间，精度到秒；可自定义一周天数、一天小时数，适合奇幻/科幻世界观。
- **叙述视图（Narrative）**：按章节/幕手动拖入事件，决定"故事讲述顺序"，与按时间排序的 Timeline 解耦，方便处理闪回、倒叙。
- **外部同步**：可与 Scrivener、Ulysses 同步，事件变动双向提醒。

**对 Plotline 的启示**：
1. Plotline 已有 timeline/gantt/tree/text 视图，但缺少**叙述编排视图**（按幕/章节组织事件）。
2. 当角色与地点增多时，关系/地铁视图能更清晰展示事件-角色-地点交集。
3. 自定义历法/相对时间是奇幻/科幻创作的高阶刚需，可作为后续版本储备。

### 2.4 Figma 社区时间轴模板：卡片分层与视觉引导

- 常见模板结构：
  - **轨道行（Track Row）**：左侧标题 + 右侧时间轴区域。
  - **事件卡片（Event Card）**：标题 + 时间标签 + 摘要，hover 展开详情/图片。
  - **连线（Connector）**：因果/伏笔用曲线或折线连接，颜色区分关系类型。
  - **标尺（Ruler）**：顶部/左侧时间刻度，支持缩放。
- 交互细节：
  - 卡片 hover 时轻微上浮（`translateY(-2px)`）+ 阴影加深。
  - 选中态使用 ring + 轻微 scale，而非高对比色块。
  - 重要节点使用"脉冲点"或"菱形标记"吸引注意力。

**对 Plotline 的启示**：
1. 事件卡片应做**信息分层**：标题 + 时间 + 状态点为一行，摘要/tag 默认隐藏或第二行显示。
2. 连线应使用**曲线/折线 + 箭头**，并在 hover 时显示关系类型。
3. 选中事件后，相关连线高亮，无关元素淡化，强化"当前焦点"。

### 2.5 Obsidian Timeline：Markdown 原生与可扩展性

- **使用方式**：通过 Markdown 标签 `timeline` / `timeline-labeled` 创建时间线，语法简单。
- **样式**：预定义 CSS 类控制线条、主体内容、活动颜色；支持自定义 CSS 变量。
- **生态**：可与其他 Obsidian 插件及本地 AI 插件结合，扩展性强。

**对 Plotline 的启示**：
1. AI 生成的内容可导出为 Markdown 时间线，方便用户迁移/分享。
2. 时间轴样式应基于 CSS 变量（`src/styles/themes.css`），便于主题切换和自定义。
3. 插件化思路：AI 能力、视图、主题都应以可扩展方式设计。

## 三、AI 助手设计惯例

### 3.1 五款 AI 创作前端对比

| 产品 | 快捷操作 | 上下文范围 | @mention / 工具调用 | Markdown 处理 | 对 Plotline 的启示 |
|---|---|---|---|---|---|
| **Claude** | 输入框下方 prompt 胶囊；Projects + Memories；Artifacts 结构化输出 | 项目文件 + 历史对话 + 用户记忆 | Claude Code `/` 命令；工具调用需审批 | 原生 Markdown，Artifact 独立渲染 | 记忆是长期创作核心；结构化输出应可编辑 |
| **Kimi** | 一键续写/润色/总结 chip；`@Kimi` 召唤 | 200万字超长上下文、联网搜索 | 支持 `@` 召唤与任务链执行 | 流式 Markdown，复制/重试按钮淡入 | 中文用户爱"一键完成"；超长上下文是杀手锏 |
| **豆包** | 桌面端划词工具栏；多场景文本生成 chip | 多轮对话 + IP 化角色 | 支持工具调用与语音对话 | 富文本渲染，表情/IP 元素多 | 划词/选中即操作，减少打断创作流 |
| **通义千问** | 一键续写/润色/公文写作 chip；浏览器插件 | 1000万字文档解析、多文档总结 | 支持多文档 `@` 引用 | 文档解析后结构化输出 | 进度可视化；提示词模板市场 |
| **GitHub Copilot Chat** | `/` slash 命令；内联补全；Agent 模式 | 当前文件 + 选中代码 + 项目索引 | `@workspace`、`@terminal` 等上下文引用 | Markdown + 代码块高亮 | 把高频操作固化为 `/` 命令 |

### 3.2 快捷操作：从"聊天框"到"能力卡片"

- **Claude**：输入框下方提供 "Help me develop a unique voice" / "Improve my writing style" 等胶囊。
- **Kimi/豆包/通义**：输入框上方固定「续写 / 润色 / 总结 / 扩写 / 翻译」等一键按钮。
- **Copilot Chat**：`/test`、`/pr-review`、`/explain` 等 slash command 把 SOP 固化。

**对 Plotline 的启示**：
1. AI 面板顶部应把能力做成**可视化卡片/chip**，而不是让用户在输入框里猜。
2. 每个能力需配**简短说明**（hint）和**适用上下文**（如"需选中事件""需在当前时间轴视图"）。
3. 引入 `/` 命令：在输入框输入 `/` 唤出能力菜单，支持键盘导航。

### 3.3 上下文范围：让用户看见 AI"看到了什么"

- **Claude Projects**：可上传文件、设定长期上下文，Claude 跨对话记住项目记忆。
- **Kimi `@` 召唤**：用户可主动引用文档或上下文。
- **Copilot Chat `@workspace`**：明确告知 AI 基于项目索引回答。

**对 Plotline 的启示**：
1. AI 面板应展示**当前上下文摘要**：工作区名称、已选实体、启用的数据源、RAG 检索到的相关条目。
2. 用户应能一键**展开/收起上下文详情**，看到"AI 参考了哪些设定"。
3. 当 AI 生成涉及冲突检测时，应**高亮冲突来源**并给出修改建议。

### 3.4 @mention / 工具调用：从对话到工作流

- **Kimi `@Kimi`**：基于当前上下文执行任务。
- **豆包桌面端划词工具栏**：选中文字后弹出悬浮条，提供翻译、润色、改写、生成大纲等操作。
- **Claude Code / Cowork**：Claude 可读取本地文件、调用工具并分步执行，用户审批后继续。

**对 Plotline 的启示**：
1. 在编辑器中**选中文字后弹出 AI 工具条**（润色、改语气、插入到时间轴）。
2. 在时间轴事件卡片右键菜单加入"AI 优化此事件""检查与此事件相关的逻辑冲突"。
3. AI 生成的内容以**Artifact 卡片**呈现，支持「插入到时间轴 / 插入到大纲 / 复制 / 弃用」。

### 3.5 Markdown 渲染：为什么星号仍会原样显示

通过对现有代码 `src/components/ui/Markdown.tsx` 的审查，发现已做了转义还原（`\*\*` → `**`），但仍有以下可能漏网：

| 问题来源 | 表现 | 解决方案 |
|---|---|---|
| 模型返回 HTML 实体 | `&ast;&ast;粗体&ast;&ast;` | 在 normalize 前解码 HTML 实体 |
| 模型返回双重转义 | `\\*\\*粗体\\*\\*` | 先 unescape 再 normalize |
| 行间标记跨行 | `**粗体\n文字**` 被 remark-breaks 拆断 | 预处理时合并跨行标记 |
| 列表项内标记被空格破坏 | `1. **设计事件类型**` 中 `**` 前有空格 | 确保 remark-breaks 不会错误拆分 |
| 流式输出中标记不完整 | `**` 被截断在两次 delta 之间 | 流式渲染时等待标记闭合再解析 |

**参考实践**：
- Claude/ChatGPT 使用**流式 Markdown 渲染器**，在收到完整 token 后逐段解析，避免半截标记。
- 多数工具会在服务端或客户端做一层**后处理**，把常见模型错误（如 `**` 被转义）统一清洗。

### 4.7 跨平台 actionable 洞察

1. **动画要稳，不要炫**：禁用弹性/弹跳/旋转 loading；退出动画不得阻塞交互。
2. **风格统一**：像素/MC 主题很有趣，但图标、色调、阴影必须遵循统一设计系统。
3. **信息密度克制**：事件卡片、AI 面板、设置页都要避免"一行塞满信息"。
4. **AI 从聊天框升级为工作流**：提供 Specialist 而非单一对话。
5. **记忆系统优先于大上下文**：建立独立设定/角色/伏笔知识库，生成前做 RAG 检索。

## 五、动画/前端效果建议

### 5.1 "炫技但不过度打扰"原则

| 原则 | 说明 | 反例 |
|---|---|---|
| **时长 150–250ms** | 微交互 150ms，视图切换 200ms | 超过 300ms 的入场动画 |
| **优先 transform/opacity** | 避免触发 layout/paint | width/height/margin 动画 |
| **ease-out 曲线** | `[0.16, 1, 0.3, 1]` 统一 | 弹性、弹跳、闪烁 |
| **可关闭/可降级** | 复杂动效应受 `prefers-reduced-motion` 与设置开关控制 | 强制播放的庆祝动画 |
| **服务核心任务** | 动画应强化用户当前操作，而非炫技 | 与当前任务无关的漂浮元素 |

### 5.2 Framer Motion 实践

- **已使用**：`AnimatePresence`、`motion.div`、拖拽约束、`useMotionValue`。
- **问题**：`AnimatePresence mode="wait"` 在某些场景下会阻塞交互；退出动画时长与入场相同，导致"拖沓"感。
- **优化建议**：
  - 入场 200ms，退出 120ms，符合"进入慢、离开快"的认知。
  - 对大量列表项使用 `layout` 动画时，加 `layoutId` 并限制数量，避免全局重排。
  - 使用 `useReducedMotion()` 统一降级。

### 5.3 GSAP 适合"炫技"时刻

- **文字拆分动画（SplitText）**：标题、空状态大标题可使用逐字/逐词淡入，但仅用于首屏或重要切换。
- **路径动画（MotionPath）**：时间轴连线可使用 GSAP 绘制 SVG 路径，hover 时高亮流动。
- **ScrollTrigger**：叙事长图导出预览时可使用，但在主界面避免过度使用。

### 5.4 CSS Houdini 与 View Transitions API

- **View Transitions API**：适合视图切换（如时间轴 ↔ 大纲 ↔ 角色），可实现"共享元素过渡"。
  - 但 WebView2 支持度需验证，建议作为渐进增强。
- **CSS Houdini**：可用于自定义背景纹理（如 MC 主题方块、纸张纹理），但不应阻塞主线程。

### 5.5 具体可落地的炫技点

| 场景 | 效果 | 技术 | 打扰度 |
|---|---|---|---|
| AI 助手首次打开 | 消息气泡依次淡入 | Framer Motion stagger | 低 |
| 完成一个番茄钟 | 方块堆叠 + 稀有矿石掉落 | Framer Motion + CSS | 中（可关闭） |
| 时间轴事件选中 | 卡片上浮 + 相关连线高亮 | Framer Motion + SVG | 低 |
| 视图切换 | 共享元素淡入淡出 | Framer Motion layoutId | 低 |
| 导出叙事长图 | 滚动路径动画 | GSAP ScrollTrigger | 高（仅在导出预览） |
| MC 主题昼夜变化 | 背景色随时间渐变 | CSS 变量 + 定时器 | 低（可关闭） |
