# v2.7.0 全面打磨与 AI 增强 — 调研报告

> 本报告汇总用户截图分析、竞品调研与中文社区反馈，为 v2.7.0 的具体实现提供依据。

---

## 一、用户截图逐像素分析

### 1.1 Markdown 星号直接显示为文本

在右侧 AI 助手面板中，AI 回复出现：

```
1. **设计事件类型** —— 是剧情驱动（选择分支）、角色驱动（情感冲突），还是环境驱动（突发事件）？
2. **添加角色** —— ...
3. **展开情节** —— ...
```

`1. **设计事件类型**` 中的 `**` 被原样渲染为文本，未转化为粗体。这说明：

- AI 消息渲染组件未启用 Markdown 解析，或解析器配置错误；
- 或者后端返回的文本被过度转义（如 `&ast;&ast;` 或 `\\*\\*`）；
- 建议词/提示词模板在生成时直接输出了裸 Markdown 标记，前端未做后处理。

### 1.2 动画与布局观察

- AI 面板消息气泡入场动画看起来拖沓，缺少明确的反馈；
- 时间轴轨道区留白过多，事件卡片与轨道标题的视觉关联不够强；
- 筛选栏按钮高度与对齐在 v2.6.2 已修复，但“今天”标签与事件卡片仍有轻微拥挤感。

## 三、中文社区反馈摘要

详见 `docs/v2.7.0-research-community.md`。核心 actionable 发现：

1. **AI 助手**：长篇创作中对话式 AI 容易“角色退化、伏笔丢失”。用户需要带「设定-角色-伏笔」记忆库的 Agent 工作流，生成前做 RAG 检索，并支持导入旧稿做风格学习。
2. **时间轴**：B站 UP 强调时间轴是“让读者跟着动线理解设计”的工具；NGA 安科创作者需要大纲与随机性平衡、多模式排序。建议支持导出叙事长图/PPT、节点锁定/开放分支。
3. **番茄钟/游戏化**：缺少自动延时、正计时、统计不与项目绑定。建议增加“自然结束”模式、自由正计时、每日目标与连续打卡，但避免排行榜焦虑。
4. **MC/像素主题**：像素字体需保证 12px/16px 整数倍清晰；仅用于标题/计时器，不用于编辑器正文；可扩展昼夜变化、方块堆叠进度等轻沉浸元素。
5. **UI/UX**：动画掉帧、风格不统一是主要槽点。动画要稳（禁用弹性/旋转 loading），复杂视图需虚拟化/降级。

## 四、v2.7.0 落地建议

基于以上调研，建议 v2.7.0 优先落地：

1. **Markdown 渲染修复**：AI 消息默认启用 Markdown 解析，模板输出增加清洗。
2. **AI 能力快捷栏**：在 AI 面板顶部固定“优化当前事件 / 优化当前片段 / 总结工作区 / 检查逻辑漏洞”四个 chip，降低用户认知成本。
3. **轻量 RAG + KV Cache**：先以关键词检索工作区实体并缓存最近会话，支撑“项目级记忆”。
4. **动画性能预算**：全局动画 150–250ms，优先 `transform`/`opacity`，减少 `layout` 动画。
5. **时间轴拖拽边界**：限制事件卡片与画布左边界，修复“拖到最左侧仍可继续拖动”。
6. **番茄钟重置与 MC 彩蛋**：增加重置成就按钮，MC 主题加入稀有矿石掉落与完成庆祝动画。
7. **字体回退栈**：得意黑/像素字体缺失时给出清晰回退，编辑器正文不使用像素字体。
