# 叙事创作工具 — 产品需求与设计文档（PRD）

> 本文档只描述功能、交互与视觉设计，不包含技术实现细节。
> 技术选型由开发团队根据当前工具能力和环境决定。

---

## 一、产品定位

### 1.1 一句话定义
一个面向小说作者、编剧和游戏叙事设计师的**本地优先**创作工作台，帮助用户通过可视化方式构建、组织和管理故事的时间线、角色关系和情节结构。

### 1.2 核心用户画像
- **小说作者**：需要管理长篇小说的时间线、角色出场顺序和伏笔回收
- **编剧**：需要可视化剧本结构，管理场景、角色弧线和三幕式节奏
- **游戏叙事设计师**：需要管理分支剧情、多轨道事件和角色关系网

### 1.3 核心使用场景
1. 创建一个新的故事项目（小说/剧本/游戏剧情）
2. 在可视化时间轴上添加、排列和编辑事件
3. 创建角色档案并关联到事件
4. 构建情节大纲（多层级：卷 → 章 → 场景）
5. 追踪伏笔和回收点
6. 导出结构化的故事文档

## 二、核心功能模块

### 2.1 工作区（Workspace）
每个项目是一个独立的工作区，包含完整的故事数据。

**功能需求**：
- 创建工作区：选择模板（空白/英雄之旅/三幕式/编年体/传记体）或从零开始
- 工作区列表：展示所有项目，支持搜索和排序（最近修改/创建时间/名称）
- 工作区设置：名称、描述、封面图、主题色
- 导入/导出：支持 JSON 格式的完整项目备份和恢复
- 删除工作区：二次确认，删除后不可恢复（本地数据）

**交互设计**：
- 启动界面：中央展示工作区卡片网格，右侧快速操作面板
- 工作区卡片：显示名称、描述、创建时间、事件数/角色数统计
- 空状态：首次使用时展示引导文案和"创建第一个项目"的醒目按钮

---

### 2.2 时间轴视图（Timeline）
故事的核心视图，以水平时间轴展示事件和轨道。

**功能需求**：
- 时间轴：水平滚动，支持缩放（年/月/日/小时级别）
- 轨道（Track）：垂直方向排列的多条平行线，每条代表一条故事线（主线/支线/角色线）
- 事件（Event）：放置在轨道上的节点，包含标题、日期、描述、关联角色
- 事件编辑：点击事件弹出详细编辑面板，支持富文本描述
- 日期系统：支持绝对日期（2024-03-15）和相对日期（故事第1天/第3章）
- 视图控制：缩放滑块、日期范围选择、轨道显隐切换
- 事件关联：通过连线表示事件之间的因果关系或伏笔回收关系

**交互设计**：
- 顶部工具栏：时间范围选择器、缩放控制、轨道管理按钮
- 左侧轨道面板：显示所有轨道列表，可重命名、改色、显隐、删除
- 画布区域：时间轴主体，支持拖拽滚动、滚轮缩放
- 事件节点：圆角矩形，显示事件标题和日期，颜色跟随轨道
- 事件详情面板：右侧滑出面板，包含标题、日期、描述、关联角色、关联事件
- 新建事件：双击画布空白处或点击"添加事件"按钮

---

### 2.3 角色管理（Characters）
管理故事中的所有角色及其关系。

**功能需求**：
- 角色列表：卡片式展示，显示头像、名字、出场次数、标签
- 角色档案：详细编辑面板，包含名字、别名、描述、外貌、背景故事、目标、冲突、角色弧线
- 角色标签：自定义分类标签（主角/配角/反派/盟友等）
- 角色关系图：可视化关系网络，节点是角色，边是关系类型（亲情/爱情/敌对/师徒等）
- 角色出场统计：在时间轴上显示该角色的出场分布
- 角色与事件关联：一个事件可以关联多个角色，一个角色可以参与多个事件

**交互设计**：
- 角色卡片：悬停显示更多信息，点击进入详情
- 关系图：力导向布局，可拖拽节点，点击边显示关系详情
- 角色筛选：按标签、出场频率、创建时间筛选

---

### 2.4 大纲视图（Outline）
树状结构展示故事的整体框架。

**功能需求**：
- 多层级结构：卷 → 章 → 场景 → 事件
- 拖放排序：支持拖拽调整章节顺序
- 大纲与事件联动：大纲中的事件节点直接关联到时间轴上的具体事件
- 状态标记：每个事件可标记状态（草稿/已完成/待修改）
- 大纲模板：支持保存常用大纲结构作为模板
- 导出大纲：支持导出为 Markdown / 纯文本格式

**交互设计**：
- 左侧树状结构：可展开/折叠的层级列表
- 右侧详情：选中节点后显示内容详情和关联事件
- 大纲轨道：每个层级可以有不同的视觉标识（颜色、图标）
- 批量操作：多选节点后批量修改状态或删除

---

### 2.5 统计视图（Statistics）
数据分析和可视化。

**功能需求**：
- 总览数据：总事件数、总角色数、总轨道数、时间跨度
- 角色出场频率：条形图/折线图展示各角色出场次数
- 情节密度热力图：时间轴上事件密度分布
- 伏笔生命周期：追踪伏笔的创建和回收时间点
- 一致性检查：检测时间矛盾（如角色同时出现在两个地点）
- 字数统计：按章节/角色/事件统计字数

**交互设计**：
- 仪表板式布局：卡片网格展示各项统计数据
- 图表交互：悬停显示具体数值，点击可筛选对应数据
- 空状态：当没有数据时显示友好的提示和引导

---

### 2.6 资料库（Notebook）
创作过程中的笔记和参考资料管理。

**功能需求**：
- 笔记文件夹：层级结构管理笔记
- 笔记编辑器：支持富文本（标题、列表、引用、代码块）
- 标签系统：为笔记添加标签便于分类
- 搜索：全文搜索笔记内容
- 笔记与工作区关联：笔记可以关联到特定工作区或全局共享
- 图片嵌入：支持插入图片作为参考素材

**交互设计**：
- 左侧文件夹树：类似文件管理器的层级结构
- 中间笔记列表：显示标题、摘要、修改时间
- 右侧编辑器：类似 Typora 的所见即所得编辑体验
- 拖拽排序：文件夹和笔记支持拖拽排序

---

### 2.7 设置（Settings）
用户偏好和全局配置。

**功能需求**：
- 主题切换：明亮/暗黑/护眼模式，支持自定义主题色
- 字体设置：编辑器字体、界面字体、字号调节
- 语言：简体中文/繁体中文/英文
- 数据管理：备份路径、自动备份频率、数据导出
- 快捷键：自定义常用操作的快捷键
- 视图偏好：默认视图、时间轴默认缩放级别、轨道默认显示状态
- 每日目标：设置每日写作字数目标，显示进度

**交互设计**：
- 标签页式布局：左侧分类导航，右侧设置项
- 实时预览：主题和字体设置实时反映在界面中
- 恢复默认：每个设置项可单独恢复默认值

## 三、数据模型

### 3.1 工作区（Workspace）
```
- id: 唯一标识
- name: 名称
- description: 描述
- template: 模板类型（blank/hero-journey/three-act/chronicle/biography）
- settings: 工作区级别的设置（主题、日期格式等）
- createdAt: 创建时间
- updatedAt: 最后修改时间
```

### 3.2 事件（Event）
```
- id: 唯一标识
- workspaceId: 所属工作区
- title: 标题
- description: 描述（支持富文本）
- date: 日期（绝对或相对）
- trackId: 所属轨道
- color: 自定义颜色（可选）
- status: 状态（draft/done/revise）
- characterIds: 关联角色列表
- connectedEventIds: 关联事件列表（伏笔/回收等）
- order: 在同轨道内的排序
- createdAt: 创建时间
- updatedAt: 最后修改时间
```

### 3.3 轨道（Track）
```
- id: 唯一标识
- workspaceId: 所属工作区
- name: 名称
- color: 颜色
- order: 显示顺序
- isVisible: 是否可见
- createdAt: 创建时间
```

### 3.4 角色（Character）
```
- id: 唯一标识
- workspaceId: 所属工作区
- name: 名称
- aliases: 别名列表
- avatar: 头像（可选）
- description: 描述
- appearance: 外貌描述
- backstory: 背景故事
- goals: 目标
- conflicts: 冲突
- arc: 角色弧线描述
- tags: 标签列表
- color: 颜色
- eventIds: 关联事件列表
- createdAt: 创建时间
- updatedAt: 最后修改时间
```

### 3.5 角色关系（CharacterRelationship）
```
- id: 唯一标识
- workspaceId: 所属工作区
- sourceId: 源角色
- targetId: 目标角色
- type: 关系类型（family/love/enemy/mentor/friend/rival等）
- description: 关系描述
- strength: 关系强度（1-5）
```

### 3.6 大纲节点（OutlineNode）
```
- id: 唯一标识
- workspaceId: 所属工作区
- type: 节点类型（volume/chapter/scene/event）
- title: 标题
- content: 内容（富文本）
- parentId: 父节点（根节点为null）
- order: 同级排序
- eventId: 关联事件（可选）
- status: 状态
- createdAt: 创建时间
- updatedAt: 最后修改时间
```

### 3.7 笔记（Note）
```
- id: 唯一标识
- workspaceId: 所属工作区（null表示全局）
- folderId: 所属文件夹
- title: 标题
- content: 内容（Markdown/富文本）
- tags: 标签列表
- createdAt: 创建时间
- updatedAt: 最后修改时间
```

## 四、交互流程

### 4.1 启动流程
1. 应用启动，显示工作区选择界面
2. 用户选择已有工作区或创建新工作区
3. 进入工作区主界面，默认显示时间轴视图

### 4.2 创建事件流程
1. 用户双击时间轴画布或点击"添加事件"按钮
2. 弹出事件编辑对话框
3. 填写标题、日期、选择轨道、添加描述
4. 关联角色（可选）：从角色列表中选择
5. 点击"保存"，事件出现在时间轴上
6. 编辑对话框关闭，时间轴自动滚动到新建事件位置

### 4.3 创建角色流程
1. 切换到角色视图
2. 点击"添加角色"按钮
3. 填写角色基本信息（名字、描述、标签）
4. 保存后角色卡片出现在列表中
5. 点击卡片可编辑详细信息

### 4.4 构建大纲流程
1. 切换到大纲视图
2. 创建卷/章结构（右键菜单或拖拽模板）
3. 在章节下添加场景节点
4. 场景节点可以关联到时间轴上的具体事件
5. 拖拽调整节点顺序和层级关系

### 4.5 导出流程
1. 用户选择导出范围（整个工作区/当前视图/选中内容）
2. 选择导出格式（Markdown / JSON / 纯文本）
3. 选择导出路径
4. 执行导出，显示进度和成功提示

## 五、视觉设计系统

### 5.1 色彩体系
- **主色**：温暖的大地色调（米白、浅棕、琥珀），避免冷色调和蓝色渐变
- **背景色**：米色/奶白色（明亮模式），深灰褐色（暗黑模式）
- **强调色**：琥珀色/陶土色（用于按钮、选中状态、高亮）
- **文字色**：深棕灰（正文），浅灰（辅助文字），白色（深色背景上）
- **轨道色**：柔和的马卡龙色系（淡粉、淡蓝、淡绿、淡黄等），避免高饱和度
- **状态色**：绿色（完成）、橙色（待修改）、灰色（草稿）

### 5.2 字体体系
- **界面字体**：无衬线字体，简洁清晰
- **编辑器字体**：等宽或衬线字体，便于长时间阅读
- **字号层级**：
  - 标题：18-20px
  - 正文：14px
  - 辅助文字：12px
  - 微小标签：10px

### 5.3 布局规范
- **应用窗口**：最小尺寸 800×600，推荐 1400×900
- **顶部工具栏**：高度 48px，包含导航、视图切换、全局搜索
- **左侧边栏**：宽度 240px，可折叠，包含功能导航
- **主画布区域**：自适应，支持水平和垂直滚动
- **右侧面板**：宽度 320px，可折叠，显示详情和属性
- **底部状态栏**：高度 32px，显示当前状态信息

### 5.4 圆角与阴影
- **卡片圆角**：8px（大卡片），4px（小元素）
- **按钮圆角**：6px
- **对话框圆角**：12px
- **阴影**：柔和弥散阴影（避免锐利、高对比度的阴影）
- **边框**：1px 浅灰边框，或无边框但用背景色区分

### 5.5 动画原则
- **过渡动画**：200-300ms，ease-out 曲线
- **页面切换**：淡入淡出，300ms
- **面板展开/收起**：滑动动画，250ms
- **拖拽反馈**：半透明预览 + 阴影增强
- **加载状态**：骨架屏或淡入脉冲，避免旋转 loading 图标
- **原则上不使用**：弹性/弹跳动画、闪烁效果、复杂的多层动画

### 5.6 图标风格
- 统一使用线性图标（outline style），2px 线宽
- 图标尺寸：16px（按钮内）、20px（导航项）、24px（空状态装饰）
- 图标颜色跟随文字颜色或主题强调色

## 六、界面设计详细说明

### 6.1 启动界面（工作区选择器）
**布局**：
- 左侧 60%：工作区卡片网格（2-3列）
- 右侧 40%：快速操作面板 + 最近项目列表
- 顶部：搜索栏 + 排序选项
- 底部：版本号、设置入口

**工作区卡片**：
- 尺寸：约 280×160px
- 内容：名称（粗体）、描述（1行截断）、创建时间、事件数/角色数
- 悬停效果：轻微上浮 + 阴影增强
- 空状态：显示引导文案和"创建第一个项目"按钮

**快速操作面板**：
- 按钮列表：新建工作区、导入项目、打开教程、设置
- 按钮样式：全宽，左侧图标 + 文字

### 6.2 时间轴视图主界面
**布局**：
- 顶部：工具栏（48px高）
- 左侧：轨道面板（可折叠，200px宽）
- 中央：时间轴画布（自适应）
- 右侧：事件详情面板（可折叠，320px宽，默认收起）

**时间轴画布**：
- 横向无限滚动，纵向固定为轨道排列
- 顶部日期标尺：按当前缩放级别显示日期刻度
- 轨道背景：交替的浅色条纹，帮助区分不同轨道
- 事件节点：圆角矩形，显示标题和日期，颜色跟随轨道
- 选中事件：外边框高亮 + 右侧面板展开
- 事件之间的连线：虚线表示伏笔，实线表示因果关系

**轨道面板**：
- 轨道列表项：色块 + 名称 + 显隐开关
- 轨道操作：重命名、改色、删除（至少保留一个轨道）
- 添加轨道按钮：在列表底部

### 6.3 角色管理界面
**布局**：
- 左侧：角色卡片网格（可切换为列表视图）
- 右侧：角色详情面板（选中后显示）
- 顶部：搜索栏 + 筛选器（标签筛选）

**角色卡片**：
- 头像（圆形或圆角矩形，默认占位图标）
- 名字
- 标签（小标签）
- 出场次数
- 悬停：显示快捷操作按钮

**角色详情面板**：
- 头像（可更换）
- 基本信息（名字、别名、描述）
- 详细档案（外貌、背景、目标、冲突、弧线）
- 关联事件列表
- 关系网络（缩略图）

### 6.4 大纲视图界面
**布局**：
- 左侧：大纲树状结构（可展开折叠）
- 右侧：选中节点的内容编辑
- 顶部：大纲操作工具栏（添加卷/章/场景、展开/折叠全部）

**树状结构**：
- 层级缩进：每层 24px
- 展开/折叠按钮：小箭头图标
- 节点类型图标：不同层级用不同图标区分
- 状态指示：节点左侧小圆点表示状态颜色
- 拖拽手柄：节点左侧的拖拽区域，用于调整顺序和层级

### 6.5 统计视图界面
**布局**：
- 仪表板式网格布局
- 每个统计项是一个卡片
- 卡片大小根据内容自适应

**卡片内容**：
- 标题 + 数值（大字号）
- 图表（条形图、折线图、饼图、热力图）
- 空状态：当没有数据时显示占位符和引导

### 6.6 设置界面
**布局**：
- 左侧：设置分类导航（200px宽）
- 右侧：设置项内容
- 分类：外观、编辑器、数据、快捷键、关于

**外观设置**：
- 主题选择：明亮/暗黑/护眼（卡片式预览）
- 主题色选择：色板选择器
- 字体设置：界面字体、编辑器字体、字号滑块
- 语言选择：下拉选择

**数据设置**：
- 备份路径：可浏览选择文件夹
- 自动备份：开关 + 时间间隔选择
- 导出数据：按钮触发导出对话框
- 导入数据：按钮触发文件选择对话框

## 七、非功能性需求

### 7.1 性能要求
- 启动时间不超过 5 秒（从点击图标到可交互）
- 时间轴视图在 1000 个事件内保持流畅滚动和缩放
- 角色关系图在 100 个角色内保持流畅交互
- 切换视图（时间轴/角色/大纲/统计）不超过 300ms
- 数据保存操作不超过 500ms

### 7.2 可靠性要求
- 所有用户操作的数据变更自动保存，无需手动点击保存
- 支持自动备份（按时间间隔，保留最近 N 个备份）
- 支持手动导出完整项目数据（JSON 格式）
- 崩溃恢复：应用异常关闭后重启可恢复工作状态
- 数据完整性：检测到数据损坏时提示用户并尝试修复

### 7.3 可用性要求
- 支持键盘快捷键操作（新建、保存、切换视图、删除等）
- 支持拖拽操作（排序、移动、关联）
- 支持撤销/重做（操作历史栈）
- 所有破坏性操作（删除）需要二次确认
- 空状态提供清晰的操作引导
- 错误提示：友好、具体，提供解决方案

### 7.4 可访问性要求
- 支持高对比度模式
- 支持键盘完全操作（无鼠标可用）
- 字体大小可调（至少支持 12-18px 范围）
- 所有图标有对应的文字标签或工具提示
- 颜色不用于传达唯一信息（配合图标或文字）

## 九、设计原则

1. **本地优先**：用户数据存储在本地，不依赖网络。即使离线也能完全使用所有功能。
2. **所见即所得**：用户操作直接反映到界面，没有隐藏的复杂流程。
3. **渐进式披露**：基础功能简单直观，高级功能通过菜单和快捷键提供，不干扰新手用户。
4. **一致性**：所有视图和组件遵循统一的视觉和交互规范。
5. **可逆性**：支持撤销/重做，破坏性操作有确认，删除有恢复可能（回收站或备份）。
6. **响应性**：界面操作即时响应，后台任务（如保存、备份）不阻塞用户操作。
7. **可定制**：用户可以根据自己的工作习惯定制界面和偏好设置。
