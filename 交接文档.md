## 已完成工作（v3.1.0 本轮迭代）

### 1. 时间轴坐标单一源重构
- **问题**：v3.0.0 前标尺、事件卡片、连接线、Today 线各自实现 `timeToX`，多源坐标计算在缩放/平移/滚动后出现错位。
- **实现**：
  - 在 `src/features/timeline/timelineGrid.ts` 新增 `ViewportState` 类型与 `getXAtTime` / `getTimeAtX` 纯函数，作为时间↔像素换算的单一源。
  - 标尺、事件卡片定位、连接线锚点、Today 线全部改用 `viewportState` + `getXAtTime` / `getTimeAtX` 计算。
  - 修复 `setTimeRange` 从 hook 解构但从未调用导致 `viewportState.timeRange` 始终为默认值的 bug。
- **状态**：✅ 已完成，新增 `timelineGrid.test.ts` 中 `getXAtTime` / `getTimeAtX` 在年/月/日/时四档 zoom 下的精度测试（误差 ≤ 1px）

### 2. 时间轴标尺 6 档刻度自适应
- **问题**：原标尺刻度档位有限，在极端 zoom 下刻度密度过高或过低，甚至崩溃。
- **实现**：
  - `chooseTickLevel(zoom)` 在 `year` / `quarter` / `month` / `week` / `day` / `hour` 之间切换，主刻度显示完整年月日。
  - 对 `0` / `NaN` / `Infinity` 极端 zoom 做兜底；修复 `chooseTickLevel` 对 `Infinity` 返回 `'year'` 而非 `'hour'` 的问题。
- **状态**：✅ 已完成

### 3. 事件卡片三 zone 重构
- **问题**：原 `EventCard` 内部布局耦合较重，时间/地点/角色信息排版拥挤。
- **实现**：
  - 拆分为 header（状态点 + 标题 + 连接句柄）/ body（时间范围 + 持续时间）/ footer（地点 + 角色头像堆叠）三 zone。
  - 时间文本统一 `tabular-nums` 与 `whitespace-normal`。
  - `Event` 类型新增 `endDateTime` / `relativeTo` / `relativeOffsetDays` 可选字段。
  - 新增 `formatEventTimeRange` 输出 "2027-12-03 14:00 – 16:00" / "相对 #3 · +2d" / "2027" 等场景化格式。
- **状态**：✅ 已完成，新增 `EventCard.test.tsx` 三 zone 结构与角色头像堆叠测试、`time.test.ts` 时间格式化测试

### 4. 连接线时间锚点边缘对齐
- **问题**：原连接线起点/终点使用事件卡片视觉中心，拖动事件后连接线偏离锚点。
- **实现**：
  - 起点/终点改为事件卡片"时间锚点边缘"：同轨道左右沿中点 / 跨轨道上下沿中点。
  - 拖动事件后连接线始终贴合锚点。
- **状态**：✅ 已完成，新增 `timeline-connection-alignment.spec.ts` 3 个 E2E 测试（同轨道/跨轨道/滚动后对齐）

### 5. 动画编排层 motionOrchestrator
- **问题**：跨视图切换、批量元素入场、拖拽归位等动画分散在各组件，stagger / delay / exit timing 不统一。
- **实现**：
  - 新增 `src/lib/motionOrchestrator.ts`，提供 5 个场景预设：`viewSwitch` / `cardBatchEnter` / `dragReturnWithConnections` / `aiPanelExpand` / `sidebarNavEnter`。
  - 支持分层缓动与 `prefers-reduced-motion` 退化；`enhanced` 关闭时统一退化为 200ms 同步淡入。
- **状态**：✅ 已完成，新增 `motionOrchestrator.test.ts`（25 个测试覆盖预设生成、enhanced 门控、stagger 计算）

### 6. 工具栏四分组 + Sidebar 三分组
- **问题**：时间轴工具栏控件散乱，Sidebar 入口重复（AI 创作 / AI Studio 并存）。
- **实现**：
  - 时间轴工具栏拆分为 Create / Filter / View mode（segmented control）/ More（下拉菜单）四分组。
  - Sidebar 重组为工作区视图 / 创作辅助 / 系统三分组，新增 12 个 `nav.tooltip.*` 文案。
  - 首次进入工作区高亮"时间轴"与"AI 创作"入口（持续 3 秒 `ring-2 ring-accent/40`）。
  - 删除 Sidebar 中重复的"AI Studio"入口，仅保留"AI 创作"。
  - 工具栏新增"导出时间轴为 Markdown"功能。
- **状态**：✅ 已完成，新增 `timeline-toolbar.spec.ts` 9 个 E2E 测试

## 上一版本迭代回顾（v2.8.1）

### 1. 时间轴事件堆叠回归修复
- **问题**：v2.8.0 中创建多个事件时，同一轨道内的事件被错误分配到瀑布式子行，导致事件整体下移、占用额外垂直空间。
- **实现**：
  - 调整 `src/features/timeline/timelineLayout.ts` 的碰撞检测与瀑布式子行逻辑，确保非真正重叠的事件保持在一行内紧凑排布。
  - 在 `src/components/views/TimelineView.tsx` 中验证渲染位置与布局计算一致。
- **状态**：✅ 已完成，新增/更新时间轴子行分配回归测试

### 2. AI 助手输出稳定化
- **问题**：AI 助手流式输出时面板高度跳动、整段重渲染导致闪烁，长回复滚动不跟手。
- **实现**：
  - 为 AI 输出面板设置最小高度，减少布局偏移。
  - 采用增量纯文本渲染，仅更新新增内容。
  - 接入平滑滚动，长回复自动跟随最新输出。
- **状态**：✅ 已完成，新增 AI 输出稳定性单元/E2E 测试

### 3. 整体 UI 清爽化
- **问题**：v2.8.0 后工具栏、侧边栏、卡片、设置页与空状态的间距和阴影风格不够统一，视觉噪音较多。
- **实现**：
  - 统一各主要视图的间距 token 与卡片阴影层级。
  - 调整设置页与空状态的留白和文案，提升长时创作舒适度。
- **状态**：✅ 已完成，视觉回归通过 Playwright 截图对比

### 4. 测试提速
- **问题**：E2E 测试中存在较多 `waitForTimeout` 硬等待，本地运行耗时较长。
- **实现**：
  - 用显式条件等待替换不必要的 `waitForTimeout`。
  - 本地 Playwright 配置启用 2 workers，充分利用多核资源。
- **状态**：✅ 已完成，全量测试在本地稳定全绿（435 单元测试 + 39 E2E 用例）
