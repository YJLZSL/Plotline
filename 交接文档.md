## 已完成工作（v2.6.1 本轮迭代）

### 1. 时间轴布局修复
- **需求**：v2.6.0 真实运行截图显示筛选栏下拉框位置偏移/截断、"今天" 标签与轨道列重叠、事件卡片内元素拥挤。
- **实现**：
  - `TimelineView.tsx` 的 `MultiSelectDropdown` 改用 `@radix-ui/react-popover` + Portal + `collisionPadding` / `avoidCollisions`，解决视口碰撞裁剪。
  - 新增 `src/features/timeline/timelineLayout.ts` 纯函数 helper，为 Today 标签提供边界保护。
  - 事件卡片最小宽度从 180px 提升至 220px；"相对" 标签与状态点合并为单一微型徽章；`+` 按钮动态避让首个事件卡片。
  - 筛选栏统一 `h-8` 高度与 `items-center` 对齐；折叠轨道标题区增加内边距与行高调整。
  - 筛选结果为空时显示 "没有匹配的事件" 空状态与清除筛选 CTA。
- **状态**：✅ 已完成，新增 `timelineLayout.test.ts` 8 个单元测试

### 2. 动画与设置页视觉重构
- **需求**：界面整体美感不足，视图切换动画不够丝滑，主题卡片布局不对称，选中态对比弱。
- **实现**：
  - `WorkspaceLayout` 与 `SettingsView` 统一使用 `AnimatePresence mode="wait"`，切换时长 200–250ms，缓动 `[0.16, 1, 0.3, 1]`。
  - 设置页主题卡片改为 2×2 网格，选中态使用 `ring-accent ring-2`。
  - 字体主题拆分为 "无衬线 / 等宽 / 得意黑 / 像素" 四个独立选项，得意黑与像素字体栈解耦；修复 `tailwind.css` / `themes.css` 变量语义冲突。
  - 设置项使用 `Card` 分组，提升扫描效率；字体主题选项改为带实时预览的按钮。
- **状态**：✅ 已完成，`SettingsView.test.tsx` 覆盖主题网格、选中态、字体主题栈

### 3. AI 助手可用性改进
- **需求**：配置好 API Key 后无法直观判断 AI 是否可用，缺少连接状态、测试连接按钮与配置摘要。
- **实现**：
  - 后端新增 `test_ai_connection` 命令（`commands/ai.rs` + `services/ai.rs`），返回状态、延迟与友好错误信息。
  - 前端新增 `testAiConnection` API 与 `useAiConnectionTest` hook。
  - 新增 `src/features/ai/components/AiSettingsSection.tsx`，在 AI 设置页顶部展示连接状态卡片与 "测试连接" 按钮。
  - API Key 输入框增加可见性切换与格式校验提示；模型下拉增加空状态/错误提示。
  - AI 面板右上角显示连接状态指示灯。
- **状态**：✅ 已完成，Rust 与前端测试覆盖连接测试逻辑

### 4. 调研与审计
- **实现**：
  - 输出 `docs/v2.6.1-ui-audit.md`，含 Playwright 真实渲染截图、问题清单、修复建议与任务映射。
  - 输出 `docs/v2.6.1-community-research.md`，汇总 B 站/V2EX/知乎/NGA 与竞品（Notion、Aeon Timeline、Claude、Kimi/豆包/通义）的设计惯例。
- **状态**：✅ 已完成

## 已完成工作（v2.6.0 本轮迭代）

### 1. AI 风格模板 chips
- **需求**：用户每次与 AI 交互都需手写长篇提示词，国产助手已教育市场「一键切换风格/题材」。
- **实现**：
  - 新增 `src/features/ai/promptTemplates.tsx`，内置玄幻/都市/言情/悬疑/番茄风/知乎风 6 种模板。
  - `AiAssistantPanel.tsx` 输入框上方新增横向 `<TemplateChips />`，点击后自动填充提示词并注入 system prompt。
  - `AiChatContext` 新增 `systemPromptOverride`，Rust `build_messages` 优先使用该覆盖值。
  - i18n 键同步到 `zh-CN.json` / `en.json`。
- **状态**：✅ 已完成，单元测试覆盖模板数量、字段与 system prompt 注入

### 2. AI apply-to-doc
- **需求**：AI 生成内容后，用户需要手动复制粘贴到小说/笔记编辑器，步骤繁琐。
- **实现**：
  - 扩展 `AiInsertTarget`，新增 `novel_chapter`、`notebook_content`、`outline_node_content`。
  - 新增 `src/stores/editorSelection.ts` Zustand store，追踪当前激活的 TipTap editor 实例与选区。
  - `NovelView`、`NotebookView`、`OutlineView` 在编辑器 focus 时注册、blur 时注销（排除切换到 AI 面板）。
  - `useApplyAiOutput` 扩展编辑器插入/替换分支；AI 回复下拉菜单按当前 view 推荐对应目标。
- **状态**：✅ 已完成，单元测试覆盖 editor selection store 与 apply-to-doc 映射

### 3. 时间轴筛选与折叠
- **需求**：事件密度高时轨道拥挤，缺少快速聚焦某条叙事线的能力。
- **实现**：
  - 新增 `src/features/timeline/useTimelineFilters.ts`，管理角色/地点/状态/搜索筛选与轨道折叠状态，并持久化到 `localStorage`。
  - `TimelineView.tsx` 顶部新增筛选栏与折叠/展开全部按钮。
  - 事件渲染前使用 `useMemo` + `filterEvents` 计算可见事件；轨道折叠时高度压缩至 24px 并显示事件数量徽标。
- **状态**：✅ 已完成，单元测试覆盖 filter 逻辑与持久化

### 4. 模板向导空状态
- **需求**：新用户创建工作区后面对空画布，不知如何开始。
- **实现**：
  - 扩展 `src/components/ui/Feedback.tsx` 的 `EmptyState`，支持 `actions` 多 CTA。
  - 新增 `src/features/workspace/seedTemplates.ts`，内置奇幻小说/都市言情/跑团战役 3 种模板数据。
  - 后端 `CreateWorkspaceInput` 增加 `template_id`，`services/workspace.rs` 在一个事务中插入模板示例数据。
  - `WorkspaceSelector.tsx` 创建弹窗新增模板选择步骤；各主要视图空状态提供「添加第一个 XX」与「从模板开始」双 CTA。
- **状态**：✅ 已完成，前后端单元测试覆盖模板创建工作区

## v2.6.2 迭代状态

### 目标

基于 v2.6.0 真实截图、浏览器测试与社区调研，修复前端错位、动画不丝滑、字体标注错误、AI 助手可用性差等体验问题；并修复 v2.6.1 中暴露的 Rust 测试编译错误，发布可用的 v2.6.2。

### 已完成工作

- 时间轴布局修复：`MultiSelectDropdown` 下拉框使用 Portal + 碰撞检测；"今天" 标签增加边界保护；事件卡片最小宽度提升并合并 "相对" 标签与状态点；筛选栏统一对齐。

- Rust 测试修复：重构 `src-tauri/src/services/ai.rs` 的测试代码，移除无法编译的 `tokio::test` 与 `reqwest::Error::from(io::Error)`，改为测试纯函数 `map_connection_test_request_error_kind` 与 `validate_connection_input`。
- 调研与审计：新增 `docs/v2.6.1-ui-audit.md`、`docs/v2.6.1-community-research.md`。
