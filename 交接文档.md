### 🔐 签名密钥问题（关键阻塞）

**问题描述**：每次发布时 CI 构建成功，但 `tauri-action` 未生成/上传 `.sig` 签名文件和 `latest.json`，导致自动更新功能失效。

**根本原因**：`TAURI_SIGNING_PRIVATE_KEY` GitHub Secret 可能未配置或已失效。Tauri 的自动更新机制要求：
1. 每个 Release 包含 `.sig` 签名文件（对应 .exe 安装器）
2. Release 包含 `latest.json` 文件（包含签名和下载 URL）
3. `src-tauri/tauri.conf.json` 中的 `pubkey` 与签名私钥匹配

**解决方案**（任选其一）：

**方案 A：重新配置签名密钥（推荐）**
1. 在本地生成新的签名密钥对：
   ```bash
   # 需要安装 Tauri CLI（cargo install tauri-cli 或 npm install -g @tauri-apps/cli）
   tauri signer generate --password="your-password"
   # 或
   npx @tauri-apps/cli@latest signer generate --password="your-password"
   ```
2. 将生成的公钥（`src-tauri/secret.pub` 的内容）更新到 `src-tauri/tauri.conf.json` 的 `pubkey` 字段。
3. 将私钥（`src-tauri/secret.key` 的内容）作为 GitHub Secret `TAURI_SIGNING_PRIVATE_KEY` 添加到仓库 Settings → Secrets → Actions。
4. 如果生成时设置了密码，将密码作为 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` 添加到 GitHub Secret。
5. 重新推送 tag 触发 CI 构建，或手动下载 .exe 后签名并上传。

**方案 B：手动签名当前 Release**
1. 下载已发布的 `Plotline_2.3.0_x64-setup.exe`。
2. 使用已有的私钥签名：
   ```bash
   tauri signer sign --private-key-path secret.key --password your-password -- Plotline_2.3.0_x64-setup.exe
   ```
3. 生成 `.sig` 文件和 `latest.json`（可用 `tauri signer sign` 的 `--output` 参数）。
4. 手动上传到 GitHub Release。

**方案 C：禁用签名验证（不推荐，安全性降低）**
- 修改 `src-tauri/tauri.conf.json`，移除 `pubkey` 字段，但这会降低自动更新的安全性。

**当前状态**：`releases/v2.3.0/latest.json` 模板已创建，但 signature 字段为 `<PLACEHOLDER>`，需要替换为真实签名后才能生效。
