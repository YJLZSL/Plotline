## [3.0.0] - 未发布

**目标**：作为 Plotline 下一重大版本，围绕“跨视图叙事导航”与“AI 创作工作流”进行整体升级。

### 新增

- **时间轴布局引擎重写** (`src/features/timeline/`):
  - 新增 `timelineGrid.ts`、`timeScale.ts`、`useTimelineViewport.ts`，统一 zoom/scroll 视口管理。
  - 重写 `timelineLayout.ts` 碰撞检测与 drag constraints，修复拖拽锚点、事件重叠回归、相对/绝对事件混合布局问题。
  - `TimelineView` 集成新布局引擎，支持以指针为锚点的缩放与画布平移。
- **独立剧本（Script）视图** (`src/components/views/ScriptView.tsx`):
  - 按故事时间线以卡片形式展示所有事件，支持显示时间、地点、关联角色与事件状态。
  - 点击任意事件跳转回时间轴并自动选中、滚动定位。
- **Timeline / Script 双向同步** (`src/stores/workspaceSelection.ts`):
  - 新增全局工作区选择状态 `useWorkspaceSelectionStore`，跨视图共享 `selectedEventId`。
  - 时间轴与剧本视图共享同一选中事件，切换视图后高亮状态保持一致。
  - `WorkspaceLayout` 在切换工作区时自动清空选择状态。
- **AI 创作助手侧边模块** (`src/features/ai-assistant/`):
  - 新增 7 个创作 Agent：对话、续写、脑暴、查漏、润色、角色关系、文风迁移。
  - 新增 Agent 选择器、上下文选择器、会话列表管理。
  - 复用现有 AI IPC 与 `mock.ts` 模拟后端，支持流式回复与会话持久化。
- **导航与路由扩展** (`src/components/layout/Sidebar.tsx` + `src/app/AppRoutes.tsx`):
  - 侧边栏新增“剧本”与“AI 创作/AI Studio”入口。
  - 路由注册 `/workspaces/:id/script` 与 `/workspaces/:id/ai-assistant`。
  - 补充中英文国际化文案 (`src/i18n/locales/zh-CN.json`、`src/i18n/locales/en.json`)。

### 优化

- **剧本视图事件排序**：排序逻辑提取到 `src/features/timeline/scriptSort.ts`，按轨道最早绝对时间锚点排序轨道，同一轨道内保持 `sortOrder` 创作顺序。
- **时间轴选中滚动定位**：从剧本视图返回时间轴时，自动将选中事件滚动到视口中央。
- **UI 清爽化延续**：统一卡片、按钮、空状态、对话框间距与阴影，降低视觉噪音。
- **Rust 代码质量**：修复 `src-tauri/src/models/ai.rs` 与 `src-tauri/src/services/ai.rs` 中 6 处已有 clippy 告警。

### 测试

- 新增 E2E 测试：
  - `tests/e2e/timeline-pan-zoom.spec.ts`：时间轴平移与缩放。
  - `tests/e2e/timeline-script-sync.spec.ts`：剧本视图与时间轴双向同步。
  - `tests/e2e/ai-assistant.spec.ts`：AI 创作助手侧边模块。
- 新增单元测试：
  - `src/features/timeline/scriptSort.test.ts`：剧本视图事件排序。
  - `src/features/timeline/timelineGrid.test.ts`、`src/lib/time.test.ts`：时间轴布局与时间工具函数。
- 全量回归：`pnpm lint`、`pnpm typecheck`、`pnpm build`、`pnpm test:run`（475 项单元测试）、`pnpm test:e2e`（51 项 E2E）全部通过。

### 发布

- 版本号统一升级到 `3.0.0`（`package.json`、`src-tauri/Cargo.toml`、`src-tauri/tauri.conf.json`、`src-tauri/Cargo.lock`）。

1. **签名密钥对**（已在 v1.3.0 生成，私钥务必安全保管，不可提交）：

   - 公钥：`keys/plotline.key.pub`（已填入 `src-tauri/tauri.conf.json`）
   - 若重新生成：`pnpm tauri signer generate -w keys/plotline.key --force`
2. **CI 机密**：把私钥内容设置到 GitHub 仓库 Settings → Secrets and variables → Actions：
   - `TAURI_SIGNING_PRIVATE_KEY` = `keys/plotline.key` 的完整内容
   - 无密码时无需设置 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`
3. **本地发布构建**（无 `pnpm` 时使用替代命令）：
   ```bash
   ./node_modules/.bin/tsc --noEmit
   ./node_modules/.bin/eslint . --max-warnings=0
   ./node_modules/.bin/vitest run
   export PATH="$PATH:/c/Users/23501/.cargo/bin"
   cargo test --manifest-path src-tauri/Cargo.toml --lib
   # 临时将 tauri.conf.json 的 beforeBuildCommand 改为 Node 直接调用后：
   ./node_modules/.bin/tauri build
   ```
4. **签名安装包并生成 latest.json**：
   ```bash
   ./node_modules/.bin/tauri signer sign --private-key-path keys/plotline.key \
     src-tauri/target/release/bundle/nsis/Plotline_1.3.0_x64-setup.exe
   # 将生成的 .sig 内容填入 releases/v1.3.0/latest.json 的 signature 字段
   ```
5. **推送标签触发 CI 发布**：
   ```bash
   git tag v1.4.0
   git push origin v1.4.0
   ```
   `.github/workflows/release.yml` 会构建 Windows 安装包并连同 `latest.json`
   一起上传到对应的 GitHub Release（应用内更新客户端轮询的就是 `latest.json`）。
   
   > 若 CI 构建失败或需要手动上传，可使用 [GitHub MCP 插件](https://github.com/cli/cli) 或 `gh` CLI 直接创建 Release 并上传产物。
6. **验证自动更新**：启动已安装的旧版本，应用在启动时会自动弹出应用内更新提示；
   也可在 *设置 → 关于 → 检查更新* 手动触发。点击「下载并安装」后应用会自动关闭并替换为新版本。
